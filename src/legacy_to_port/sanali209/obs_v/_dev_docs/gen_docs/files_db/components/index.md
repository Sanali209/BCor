# Компоненты модуля `files_db`

Этот раздел описывает основные строительные блоки модуля `files_db`, отвечающие за представление и управление различными аспектами данных о файлах.

## Подразделы

*   [Каталоги](./catalogs.md) - Управление каталогами или коллекциями файлов.
*   [Связи](./relations.md) - Управление связями между файлами (например, поиск дубликатов, эмбеддинги).
*   [Веб-ссылки](./web_links.md) - Управление ссылками на внешние веб-ресурсы, связанные с файлами.

## Взаимодействие компонентов

Компоненты модуля `files_db` работают совместно для обогащения и связывания информации о файлах:

*   **Каталоги (`catalogs.py`)**: Представляют собой механизм для логической группировки файлов (хотя их текущее использование может быть ограничено).
*   **Связи (`relations.py`, `dubsearch.py`)**: Используют [векторный поиск](./../vector_db.md) для нахождения семантически похожих элементов (файлов, детекций) на основе их эмбеддингов. Результаты сохраняются как записи `RelationRecord`, связывая исходные элементы в MongoDB.
*   **Веб-ссылки (`web_link_record.py`)**: Управляют записями `WebLinkRecord`, представляющими внешние URL. Они могут быть связаны с файлами (например, как источник) и содержать дополнительную информацию, такую как превью или краткое содержание, сгенерированное с помощью AI-моделей.

Эти компоненты взаимодействуют с основными [классами-обертками](./../core_concepts.md#ключевые-классы-обертки) (например, `FileRecord`) и [индексаторами](./../indexers/index.md) для получения и сохранения данных.
