# Компонент: Связи

Этот компонент отвечает за управление связями между записями (в основном, файлами), включая поиск дубликатов и работу с эмбеддингами.

## Базовый класс `RelationRecord` (`relation.py`)

*   **Назначение**: Представляет направленную связь между двумя записями (`CollectionRecord`) в базе данных MongoDB.
*   **Коллекция MongoDB**: `relation_records`.
*   **Наследование**: Наследуется от `SLM.mongoext.wraper.MongoRecordWrapper`.
*   **Ключевые поля MongoDB**:
    *   `_id`: Уникальный идентификатор.
    *   `type` (str): Тип связи (например, "duplicate", "similarity").
    *   `sub_type` (str): Подтип связи (например, "md5", "CLIP_embedding").
    *   `from_id` (ObjectId): Ссылка на `_id` исходной записи.
    *   `to_id` (ObjectId): Ссылка на `_id` целевой записи.
    *   *(Другие поля, такие как `distance`, могут добавляться для специфических типов связей, судя по индексам, создаваемым в `files_db_module.py`)*.
*   **Основные методы**:
    *   `get_or_create(from_, to_, type, **kwargs)`: Находит или создает связь.
    *   `set_relation(from_, to_, type_)`: Псевдоним для `get_or_create`.
    *   `is_exist(from_, to_, type_)`: Проверяет, существует ли связь.
    *   `get_outgoing_relations(from_CollectionRecord, type_)`: Возвращает список исходящих связей заданного типа для объекта.
    *   `delete_all_relations(obj)`: Статический метод, удаляющий все связи, где `obj._id` указан в `from_id` или `to_id`. Используется для каскадного удаления при удалении `CollectionRecord`.

## Другие модули

*   **Поиск схожести (`dubsearch.py`)**:
    *   **Назначение**: Этот модуль реализует поиск **похожих** файлов (`FileRecord`) или распознанных объектов (`Detection`, таких как лица) на основе векторных эмбеддингов, а не только поиск точных дубликатов по хэшу.
    *   **Интеграция с `VectorDB`**:
        *   Регистрирует множество пресетов в `VectorDB` для различных моделей эмбеддингов (FaceNet, ResNet, CLIP, DINO, BLIP и др.), как для объектов (`face`, `person`), так и для файлов (`FileRecord_*`).
        *   Использует `VectorDB.SearchScopeList` для выполнения поиска схожести в пределах набора данных или по отношению к другому набору.
    *   **Рабочие процессы**:
        1.  **Поиск похожих объектов (`create_face_graph`)**: Находит похожие `Detection` (лица, люди) и создает между ними `RelationRecord` с `type="similar_obj_search"`.
        2.  **Проекция на файлы (`project_object_to_image_graps`)**: Переносит найденную схожесть объектов на их родительские файлы, создавая/обновляя `RelationRecord` с `type="similar_search"` и `sub_type`, указывающим на тип объекта ('face', 'person').
        3.  **Прямой поиск похожих файлов (`create_graf_dubs`)**: Находит похожие `FileRecord` напрямую по их эмбеддингам и создает `RelationRecord` с `type="similar_search"` и `sub_type="none"`.
    *   **Использование `RelationRecord`**:
        *   `type`: `"similar_search"` (для файлов) или `"similar_obj_search"` (для объектов).
        *   `sub_type`: `"none"` (прямое сравнение), `"face"`, `"person"` (проекция со схожести объектов).
        *   `emb_type`: Имя использованного пресета/модели.
        *   `distance`: Расстояние между векторами (мера схожести).
    *   *Примечание: Поиск точных дубликатов по MD5, вероятно, выполняется при добавлении файлов (`FileRecord.create_record_data`) или в другом месте.*
    *   **Создание связей**: Функции в `dubsearch.py` (`create_face_graph`, `project_object_to_image_graps`, `create_graf_dubs`) выполняют поиск схожести с помощью `vector_db.py` и затем **создают соответствующие `RelationRecord`** типа `"similar_search"` или `"similar_obj_search"`.
*   **Эмбеддинги (`embeddings/`)**:
    *   Эта папка содержит подпапки для различных моделей эмбеддингов (BLIP, CLIP, DINO и т.д.), но не содержит исполняемого кода для создания связей.
    *   Логика получения эмбеддингов находится в `files_db_module.py` (функции `vectorize_*`) и использует `Embeddings_cache`.
