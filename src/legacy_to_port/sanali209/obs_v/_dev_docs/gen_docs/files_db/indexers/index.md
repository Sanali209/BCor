# Индексаторы модуля `files_db`

Этот раздел описывает систему индексаторов, используемую для извлечения метаданных и другой информации из файлов и сохранения ее в записях `FileRecord`.

## Архитектура (`SLM.indexerpyiplain.indexpyiplain`)

Система построена на двух основных классах:

1.  **`ItemIndexer`**:
    *   Представляет конвейер (pipeline) для индексации одного элемента (`item`, обычно `FileRecord`).
    *   Содержит словарь индексаторов полей (`field_indexers`).
    *   Индексаторы добавляются через метод `add_field_indexer` или оператор `|`.
    *   Метод `index(item, reindex=False)` запускает конвейер:
        *   Проверяет необходимость запуска каждого индексатора (`isNeedIndex`).
        *   Управляет зависимостями между индексаторами (`invoce_dependent_fields`).
        *   Вызывает метод `index` конкретного индексатора.
    *   Позволяет обмениваться данными между индексаторами через `shared_data`.

2.  **`ItemFieldIndexer`**:
    *   Базовый класс для конкретного индексатора. Каждый подкласс отвечает за извлечение и сохранение определенного типа информации (например, EXIF, теги из имени, MD5).
    *   **Ключевые атрибуты/методы для переопределения**:
        *   `fieldName` (str): Уникальное имя индексатора.
        *   `enabled` (bool): Включен ли индексатор.
        *   `dependent_fields` (list): Список имен индексаторов-зависимостей.
        *   `isNeedIndex(parent_indexer, item)`: Определяет, нужно ли запускать индексатор для данного `item`.
        *   `index(parent_indexer, item, need_index)`: Реализует логику извлечения и сохранения данных в `item`.

Эта архитектура позволяет создавать гибкие и расширяемые конвейеры индексации с управлением зависимостями.
