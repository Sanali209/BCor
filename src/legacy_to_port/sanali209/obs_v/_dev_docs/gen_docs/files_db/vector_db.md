# Векторная база данных (`vector_db.py`)

Этот модуль предоставляет абстракцию для **in-memory** поиска по сходству векторов с использованием библиотеки `Annoy`. Он позволяет находить похожие элементы (например, `FileRecord`, `Detection`) на основе их векторных эмбеддингов.

*Примечание: Файл `vector_db_ext.py`, упомянутый ранее, не найден. Функционал реализован в `vector_db.py`.*

## Основные компоненты

1.  **`VectorDB` (Статический класс)**:
    *   **Реестр конфигураций**: Управляет различными "предпочтениями" (`VectorDBPreferences`) для создания векторных индексов.
    *   **`register_pref(name, vector_size, vector_function, metric_name)`**: Регистрирует новую конфигурацию, указывая:
        *   `name`: Уникальное имя (например, "FileRecord_CLIP", "face_FaceNet").
        *   `vector_size`: Размерность вектора эмбеддинга.
        *   `vector_function`: Функция, принимающая элемент и возвращающая его вектор (например, функции `vectorize_*` из `files_db_module.py`).
        *   `metric_name`: Метрика расстояния `Annoy` ("angular", "euclidean", "manhattan", "hamming", "dot").
    *   **`get_pref(name)`**: Возвращает объект `VectorDBPreferences` для данного имени (ленивое создание).

2.  **`VectorDBPreferences`**:
    *   Класс данных, хранящий параметры конфигурации (`name`, `vector_function`, `metric_name`, `vector_size`).

3.  **`SearchScope` (Абстрактный базовый класс)**:
    *   Представляет построенный **in-memory** индекс `Annoy` для набора элементов.
    *   **`__init__(db_table: VectorDBPreferences)`**:
        *   Получает список элементов через `get_items_to_vectorization()` (реализуется в подклассах).
        *   **Векторизует** все элементы с помощью `vector_function`.
        *   **Строит индекс `Annoy`** в памяти.
    *   **`search(search_term, limit, distance_threshold)`**: Находит `limit` ближайших соседей для `search_term` с расстоянием не больше `distance_threshold`.
    *   **`find_dubs(limit, distance_threshold)`**: Находит "дубликаты" (ближайших соседей) для *каждого* элемента в индексе.
    *   **`get_items_to_vectorization()`**: Абстрактный метод для получения списка элементов.

4.  **`SearchScopeList(SearchScope)`**:
    *   Конкретная реализация `SearchScope`, принимающая список элементов (`items_list`) в конструкторе.

5.  **`ResultItem` / `ResultGroup`**: Классы для структурирования результатов поиска.

## Принцип работы

1.  **Регистрация**: Различные способы векторизации (например, CLIP для файлов, FaceNet для лиц) регистрируются через `VectorDB.register_pref` (см. `dubsearch.py`).
2.  **Индексация и Поиск**:
    *   Получается нужная конфигурация (`VectorDBPreferences`) через `VectorDB.get_pref`.
    *   Создается экземпляр `SearchScopeList`, передавая ему конфигурацию и список элементов (например, `FileRecord.find(...)`). **На этом этапе происходит векторизация всех элементов и построение индекса Annoy в памяти.**
    *   Вызываются методы `search` или `find_dubs` у объекта `SearchScopeList`.
    *   Результаты поиска (объекты `ResultGroup`) используются для создания записей `RelationRecord` (см. `dubsearch.py`).

## Ограничения

*   **In-memory**: Индекс строится и хранится полностью в оперативной памяти, что может быть неэффективно для очень больших наборов данных.
*   **Статичность**: Индексы `Annoy` статичны. При добавлении/удалении данных необходимо перестраивать индекс (создавать новый `SearchScope`).
