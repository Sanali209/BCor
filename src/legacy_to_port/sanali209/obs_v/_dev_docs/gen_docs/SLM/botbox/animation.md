# Animation System (`animation.py`)

Этот модуль реализует базовый класс для управления спрайтовой анимацией в игре.

## Класс `Animation`

```python
class Animation:
    def __init__(self, sheet, frame_rects, frame_duration):
        # ... реализация ...
```

### Назначение

Класс предоставляет функционал для:
- Извлечения кадров анимации из спрайтового листа
- Управления временем показа каждого кадра
- Циклического воспроизведения последовательности кадров

### Конструктор

```python
def __init__(self, sheet, frame_rects, frame_duration):
```

**Параметры:**
- `sheet`: Спрайтовый лист (изображение, содержащее все кадры)
- `frame_rects`: Список прямоугольников `(x, y, width, height)`, определяющих положение каждого кадра на листе
- `frame_duration`: Длительность показа одного кадра в секундах

**Действия:**
1. Извлекает кадры из спрайтового листа с помощью функции `get_frame`
2. Сохраняет длительность показа кадра
3. Инициализирует счетчики текущего кадра и времени

### Атрибуты

- `frames`: Список извлеченных кадров анимации
- `frame_duration`: Время показа одного кадра
- `current_frame`: Индекс текущего отображаемого кадра
- `time_accumulator`: Накопитель времени для определения момента смены кадра

### Методы

#### `update(self, dt)`

```python
def update(self, dt):
    # ... реализация ...
```

**Назначение:** Обновляет состояние анимации на основе прошедшего времени.

**Параметры:**
- `dt`: Время, прошедшее с предыдущего обновления (в секундах)

**Алгоритм:**
1. Добавляет прошедшее время к аккумулятору
2. Если накоплено больше времени чем `frame_duration`:
   - Вычитает `frame_duration` из аккумулятора
   - Переходит к следующему кадру (с зацикливанием)

#### `get_image(self)`

```python
def get_image(self):
    # ... реализация ...
```

**Назначение:** Возвращает текущий кадр анимации.

**Возвращаемое значение:** Изображение текущего кадра.

### Детали реализации

1. **Циклическое воспроизведение:**
   - Использует операцию `% len(self.frames)` для зацикливания анимации
   - Нет явного состояния воспроизведения (всегда циклическое)

2. **Накопление времени:**
   - Позволяет корректно работать с переменным `dt`
   - Избегает проблем с пропуском кадров при низком FPS

3. **Зависимости:**
   - Требуется внешняя функция `get_frame` для извлечения кадров из спрайтового листа
   - Предполагается совместимость форматов изображений с системой рендеринга

### Ограничения

1. Отсутствует возможность:
   - Приостановки/возобновления анимации
   - Воспроизведения в обратном порядке
   - Единичного воспроизведения (без зацикливания)
   - Изменения скорости воспроизведения на лету

2. Нет проверок:
   - Корректности входных параметров
   - Существования кадров
   - Положительности `frame_duration`

### Возможные улучшения

1. **Функциональность:**
   - Добавление состояний (играет/пауза)
   - Поддержка обратного воспроизведения

   - Callback при завершении цикла

2. **Безопасность:**
   - Валидация входных параметров
   - Обработка краевых случаев
   - Защита от отрицательного `dt`

3. **Производительность:**
   - Кэширование извлеченных кадров
   - Ленивая загрузка кадров
   - Оптимизация памяти
