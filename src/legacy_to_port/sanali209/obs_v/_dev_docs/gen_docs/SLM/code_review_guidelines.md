# Руководство по проверке кода SLM

Этот документ содержит рекомендации по проверке кода для проекта SLM, обеспечивающие качество, согласованность и поддерживаемость кодовой базы.

## Основные принципы

### 1. Ясность и читаемость
- Код должен быть самодокументирующимся
- Используйте говорящие имена переменных и функций
- Добавляйте комментарии для сложной логики
- Следуйте принципу DRY (Don't Repeat Yourself)

### 2. Корректность
- Код должен решать поставленную задачу
- Должны быть обработаны краевые случаи
- Входные данные должны проверяться
- Ошибки должны обрабатываться правильно

### 3. Эффективность
- Оптимальное использование ресурсов
- Масштабируемость решения
- Избегание избыточных операций
- Правильный выбор структур данных

### 4. Согласованность
- Соответствие PEP 8
- Следование архитектуре проекта
- Единый стиль кодирования
- Согласованное именование

### 5. Тестируемость
- Модульные тесты для нового кода
- Интеграционные тесты при необходимости
- Тесты производительности для критичных частей
- Документация тестовых сценариев


## Процесс проверки

### 1. Подготовка PR
- Понятное описание изменений
- Ссылки на задачи/issues
- Список основных изменений
- Инструкции по тестированию


### 3. Детальная проверка
1. Изучение кода
2. Запуск тестов
3. Проверка производительности
4. Проверка документации

### 4. Обратная связь
- Конструктивные комментарии
- Предложения по улучшению
- Ссылки на документацию/примеры
- Пояснение проблем

## Стандарты кода

### Форматирование
```python
# Правильно
def process_data(input_data: Dict[str, Any]) -> List[Result]:
    """Обработка входных данных.
    
    Длинное описание метода...
    """
    if not input_data:
        return []
```

### Именование
```python
# Правильно
class UserAccountManager:
    def create_new_account(self, username: str) -> Account:
        pass

# Неправильно
class UAM:
    def cna(self, un):
        pass
```

### Комментарии
```python
# Правильно
def complex_algorithm():
    # Шаг 1: Подготовка данных
    data = prepare_data()
    
    # Шаг 2: Основные вычисления
    result = perform_calculations(data)
    
    # Шаг 3: Валидация результата
    validate_result(result)
```

## Автоматизация
