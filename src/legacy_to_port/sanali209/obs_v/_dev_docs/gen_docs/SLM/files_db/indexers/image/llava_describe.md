# LLaVA Image Description Indexer (`llava_describe.py`)

Этот модуль реализует индексатор для автоматического создания текстовых описаний изображений с помощью модели LLaVA (mc_llava_13b_4b).

## Зависимости

*   `SLM.files_db.indexers.image.content_md5.files_db_indexer`: Базовый класс для индексаторов.
*   `SLM.chains.chains_main`: Утилиты для работы с цепочками обработки (импортируются, но не используются в текущей реализации).
    *   `DictFieldMergeChainFunction`
    *   `DictFormatterChainFunction`
*   `SLM.groupcontext.group`: Импортируется, но не используется в текущей реализации.
*   `SLM.indexerpyiplain.idexpyiplain.ItemIndexer`: Базовый интерфейс индексатора.
*   `SLM.metadata.MDManager.mdmanager.MDManager`: Импортируется, но не используется в текущей реализации.

## Класс `ImageLLavaDescribe`

```python
class ImageLLavaDescribe(files_db_indexer):
    # ... реализация ...
```

### Инициализация

*   **`__init__`:**
    *   Устанавливает `fieldName = "ImageLLavaDescribe"`.
    *   Этот `fieldName` используется для отслеживания, был ли файл уже обработан этим индексатором.

### Основная функциональность

**`index(self, parent_indexer: ItemIndexer, item, need_index)`:**

1.  **Получение FileRecord:**
    *   Создает объект `FileRecord` для доступа к файлу изображения по `item["_id"]`.

2.  **Генерация описания:**
    *   Вызывает `file_item.get_ai_expertise()` со следующими параметрами:
        *   `"image-text"`: Тип экспертизы (анализ изображения с генерацией текста).
        *   `"mc_llava_13b_4b"`: Используемая модель (LLaVA 13B).
        *   `question="detailed describe image"`: Промпт для модели.

3.  **Сохранение результата:**
    *   Обновляет `file_item.description` результатом из `result['data']`.
    *   Устанавливает флаг `parent_indexer.shared_data["item_indexed"] = True`.
    *   Вызывает `self.mark_as_indexed()` для отметки записи как обработанной.

## Использование

Этот индексатор является частью цепочки индексации изображений и отвечает за:
1.  Генерацию подробных текстовых описаний изображений с помощью модели LLaVA.
2.  Сохранение этих описаний в поле `description` файловой записи.

Это позволяет:
*   Получать семантически богатые описания содержимого изображений.
*   Использовать эти описания для полнотекстового поиска.
*   Потенциально использовать их для более сложного анализа или категоризации (с помощью дополнительных индексаторов).

## Отличительные особенности

1.  **Использование LLaVA:**
    *   Использует мощную мультимодальную модель для понимания содержимого изображений.
    *   Генерирует естественные текстовые описания вместо простых тегов.

2.  **Простота интеграции:**
    *   Использует существующую систему AI-экспертиз через `get_ai_expertise()`.
    *   Требует минимальной настройки (только промпт "detailed describe image").
