# Embeddings (`embeddings/`)

Эта директория внутри пакета `relations` предназначена для организации ресурсов, связанных с различными моделями, используемыми для генерации векторных представлений (эмбеддингов). Эмбеддинги являются основой для поиска визуальной схожести, реализованного в модуле `dubsearch.py`.

## Структура

Директория содержит поддиректории, каждая из которых соответствует конкретной модели или архитектуре нейронной сети, используемой для получения эмбеддингов:

*   `BLIP/`: Ресурсы для модели BLIP.
*   `CLIP_DML/`: Ресурсы для модели CLIP с Deep Metric Learning.
*   `CNN_Encoder_FaceNet/`: Ресурсы для модели FaceNet (вероятно, для эмбеддингов лиц).
*   `DINO/`: Ресурсы для модели DINO (Self-**di**stillation with **no** labels).
*   `InceptionResNetV2/`: Ресурсы для модели InceptionResNetV2.
*   `InceptionV3/`: Ресурсы для модели InceptionV3.
*   `ModileNetV3Small/`: Ресурсы для модели MobileNetV3-Small.
*   `ResNet50/`: Ресурсы для модели ResNet50.

*(Примечание: Содержимое этих поддиректорий не было детализировано, но они могут содержать файлы весов моделей, конфигурационные файлы или специфический код загрузки/инициализации.)*

## Назначение

*   **Организация ресурсов:** Централизованное хранение файлов, необходимых для работы конкретных моделей эмбеддинга.
*   **Модульность:** Позволяет легко добавлять или обновлять поддержку новых моделей, размещая их ресурсы в соответствующих поддиректориях.

Код, непосредственно использующий эти модели для генерации эмбеддингов, вероятно, находится в других местах, таких как:
*   Модули энкодеров в `SLM/vision/imagetotensor/backends/` и `SLM/vision/imagetotensor/custom/`.
*   Функции векторизации в `SLM/files_db/files_db_module.py`.
*   Система поиска схожести в `SLM/files_db/components/relations/dubsearch.py`.
