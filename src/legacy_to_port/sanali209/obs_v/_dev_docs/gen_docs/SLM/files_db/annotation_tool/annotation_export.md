# Annotation Data Export (`annotation_export.py`)

Этот модуль предоставляет инфраструктуру и конкретные реализации для экспорта аннотированных данных из `AnnotationJob` в форматы, подходящие для обучения моделей машинного обучения. Он включает менеджер экспортеров и несколько стратегий экспорта для задач классификации изображений.

## Зависимости

*   `os`, `uuid`, `collections.defaultdict`, `typing.List`: Стандартные библиотеки Python.
*   `loguru`, `tqdm`: Внешние библиотеки для логирования и прогресс-баров.
*   `PIL (Pillow)`: Для работы с изображениями (загрузка, проверка, изменение размера, сохранение).
*   **SLM Core & Utils:** `SLM.appGlue.core` (`Service`, `Allocator`), `SLM.appGlue.iotools.pathtools.copy_file_ifExist`.
*   **SLM Cache:** `SLM.files_data_cache.pool.PILPool`.
*   *(Неявная зависимость от `SLM.files_db.annotation_tool.annotation` для типов `AnnotationJob`, `AnnotationRecord` в аннотациях типов и докстрингах)*.

## Вспомогательные функции

*   **`load_md5_list(md5_file_path)`**: Загружает список MD5 хэшей из файла (предположительно, для отслеживания уже экспортированных файлов). Возвращает `set`.
*   **`save_md5_list(md5_list, md5_file_path)`**: Сохраняет список MD5 хэшей в файл, перезаписывая существующий.
*   **`get_last_batch_number(dataset_path)`**: Находит и возвращает номер последней папки-батча (папки с числовым именем) в указанном пути датасета. Используется для инкрементального экспорта.
*   **`create_batch(bach_size, distribution, labeled_images)`**: Формирует батч изображений заданного размера (`bach_size`), пытаясь соблюсти заданное распределение классов (`distribution`). Удаляет выбранные изображения из исходного словаря `labeled_images`.

## Базовый класс `DataSetExporter`

*   **Назначение:** Абстрактный базовый класс для всех экспортеров датасетов.
*   **Атрибуты:**
    *   `name` (str): Уникальное имя экспортера.
*   **Методы:**
    *   `is_job_type_supported(self, job_type: str)`: Должен быть переопределен. Возвращает `True`, если экспортер поддерживает данный тип задания аннотирования (`AnnotationJob.type`).
    *   `ExportToDataset(self, dataset_path: str, annotation_job)`: Должен быть переопределен. Основной метод, выполняющий логику экспорта данных из `annotation_job` в `dataset_path`.

## Менеджер Экспортеров (`DataSetExporterManager`)

*   **Назначение:** Сервис (`Service`), зарегистрированный в `Allocator`, который управляет доступными экземплярами `DataSetExporter`.
*   **Атрибуты:**
    *   `exporters` (List[`DataSetExporter`]): Список зарегистрированных экспортеров.
*   **Методы:**
    *   `register(self, exporter: DataSetExporter)`: Добавляет новый экспортер в список.
    *   `get_exporter_by_name(self, name: str)`: Находит и возвращает экспортер по имени.
    *   `get_all_exporters()`: Возвращает список всех зарегистрированных экспортеров.
    *   `get_all_supported_job_types(self, job_type: str)`: Возвращает список экспортеров, поддерживающих указанный тип задания.

## Конкретные реализации экспортеров

### 1. `DataSetExporterImageMultiClass_dirs(DataSetExporter)`

*   **Имя:** `"ImageMultiClass_dirs"`
*   **Поддерживаемый тип задания:** `"multiclass/images"` (Вероятно, опечатка, должно быть "multiclass/image" как в `annotation.py`)
*   **Логика экспорта (`ExportToDataset`):**
    1.  Получает все аннотированные записи (`AnnotationRecord`) из `annotation_job`.
    2.  Для каждой записи:
        *   Создает поддиректорию в `dataset_path`, имя которой соответствует метке (`item.value`).
        *   Проверяет, существует ли файл изображения (`item.file.full_path`).
        *   Проверяет, не превышен ли лимит (`self.category_limit`, по умолчанию 100000) для данной категории.
        *   Загружает изображение с помощью `PILPool`, проверяет его целостность (`verify()`).
        *   Генерирует уникальное имя файла (UUID).
        *   Создает миниатюру изображения (256x256).
        *   Сохраняет миниатюру в формате JPG в соответствующую поддиректорию класса.
    *   **Структура датасета:** Папка для каждого класса, содержащая изображения этого класса.

### 2. `DataSetExporterImageMultiClass_dirs_cum(DataSetExporter)`

*   **Имя:** `"ImageMultiClass_dirs"` (То же имя, что и у предыдущего, что может вызвать конфликт при поиске по имени. Вероятно, имя должно быть уникальным, например, `"ImageMultiClass_dirs_cumulative"`).
*   **Поддерживаемый тип задания:** `"multiclass/images"` (Та же вероятная опечатка).
*   **Логика экспорта (`ExportToDataset`):** Реализует **инкрементальный (кумулятивный)** экспорт с разбивкой на батчи.
    1.  Загружает список MD5 уже экспортированных файлов (`load_md5_list`).
    2.  Определяет номер последнего созданного батча (`get_last_batch_number`).
    3.  Получает все аннотированные записи из `annotation_job`.
    4.  Фильтрует записи:
        *   Пропускает записи без файла или с несуществующим файлом.
        *   Пропускает файлы, MD5 которых уже есть в загруженном списке.
        *   Пропускает файлы, относящиеся к игнорируемым категориям (`ignore_cat`).
        *   Ограничивает количество файлов на категорию (`self.clamp_cat`, по умолчанию 10000).
        *   Собирает отфильтрованные `FileRecord` в словарь `labeled_images` по категориям.
        *   Добавляет MD5 новых файлов в `md5_list`.
    5.  Рассчитывает желаемое распределение классов (`distribution`).
    6.  В цикле:
        *   Формирует батч изображений (`create_batch`), пытаясь соблюсти распределение.
        *   Если батч пуст, выходит из цикла.
        *   Создает новую папку для батча (с инкрементированным номером).
        *   Для каждого изображения в батче:
            *   Определяет его метку.
            *   Создает подпапку для метки внутри папки батча.
            *   Загружает, проверяет, создает миниатюру (256x256) и сохраняет изображение в папку метки внутри папки батча.
    7.  Сохраняет обновленный список MD5 (`save_md5_list`).
    *   **Структура датасета:** Папки с номерами батчей (`1`, `2`, ...), внутри каждой - папки классов с изображениями. Файл `data.json` (вероятно, должен быть `data.txt` или другое имя, судя по коду `load/save_md5_list`) хранит MD5 всех экспортированных файлов.

### 3. `DataSetExporterImageMultiClass_anomali(DataSetExporter)`

*   **Имя:** `"ImageMultiClass_dirs"` (Опять то же имя).
*   **Поддерживаемый тип задания:** `"multiclass/images"` (Та же вероятная опечатка).
*   **Логика экспорта (`ExportToDataset`):** Похоже, предназначен для экспорта изображений, которые **не удовлетворяют** некоторому критерию, заданному объектом `Annotator` (передается как аргумент).
    1.  *(Странный шаг)*: Копирует по одному изображению для каждого класса в соответствующие папки. Возможно, для инициализации структуры папок или как базовый пример.
    2.  Получает все аннотированные записи.
    3.  Для каждой записи:
        *   Проверяет существование файла.
        *   Использует `Annotator.is_satisfied_by(item.value, item.file)` для проверки.
        *   Если `is_satisfied_by` возвращает `False` (т.е. это "аномалия" или несоответствие):
            *   Создает папку для класса, если ее нет.
            *   Копирует **оригинальный** файл (без изменения размера) в папку класса.
    *   **Структура датасета:** Папки классов, содержащие оригинальные изображения, которые не прошли проверку `Annotator`.

**Регистрация:** Экземпляр `DataSetExporterImageMultiClass_dirs_cum` регистрируется в `DataSetExporterManager` при загрузке модуля. Другие экспортеры не регистрируются автоматически.
