# Package Initializer (`__init__.py`)

Этот файл `__init__.py` находится в корне пакета `files_db` и выполняет ключевую роль в интеграции этого пакета с основной системой модулей приложения (`appGlue`).

## Назначение

Основная задача этого файла - **автоматическая регистрация** основного модуля `FilesDBModule` при импорте пакета `files_db`.

## Реализация

1.  **Импорты:**
    *   `from SLM import Allocator`: Импортирует глобальный объект `Allocator`, который отвечает за управление модулями и зависимостями в приложении.
    *   `from SLM.files_db.files_db_module import FilesDBModule`: Импортирует класс `FilesDBModule` из соседнего файла `files_db_module.py`.
2.  **Регистрация модуля:**
    *   `Allocator.add_module(FilesDBModule())`: Создает экземпляр `FilesDBModule` и немедленно передает его методу `add_module` объекта `Allocator`.

## Эффект

Благодаря этому коду, как только любая часть приложения выполнит импорт пакета `files_db` (например, `import SLM.files_db` или `from SLM.files_db import ...`), экземпляр `FilesDBModule` будет создан и зарегистрирован в `Allocator`. Это гарантирует, что метод `FilesDBModule.load()` будет вызван на соответствующем этапе инициализации приложения, обеспечивая настройку базы данных, кэша эмбеддингов, регистрацию моделей и создание индексов до того, как они понадобятся другим компонентам.
