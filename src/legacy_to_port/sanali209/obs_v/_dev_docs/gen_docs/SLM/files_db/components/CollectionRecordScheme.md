# File Type Schemes and Router (`CollectionRecordScheme.py`)

Этот модуль определяет систему для управления конфигурациями обработки различных типов файлов, особенно в контексте их индексации. Он вводит понятие "схемы файла" (`FileScheme`) и предоставляет "маршрутизатор" (`FileTypeRouter`) для выбора подходящей схемы на основе типа файла.

## Зависимости

*   `os`, `fnmatch`, `copy`
*   `SLM.appGlue.core.Service`, `SLM.appGlue.core.Allocator`
*   Различные классы индексаторов из `SLM.files_db.indexers.image.*`
*   `SLM.indexerpyiplain.idexpyiplain.ItemIndexer`

## Основные классы

### `FileScheme(Service)`

Абстрактный базовый класс для определения конфигурации (схемы) для конкретного типа файла.

*   **Назначение:** Хранить метаданные о типе файла и связанный с ним конвейер индексации.
*   **Атрибуты:**
    *   `name` (str): Уникальное имя схемы (например, `"Image:JPG"`).
    *   `mach_patterns` (list[str]): Список шаблонов расширений файлов (например, `[".jpg", ".jpeg"]`), используемых `FileTypeRouter` для сопоставления.
    *   `content` (str): Общий тип содержимого (например, `"Image"`).
    *   `attachments` (dict): Словарь для хранения произвольных связанных данных. Ключевым является `"base_indexer"`, где хранится настроенный конвейер `ItemIndexer`.
*   **Методы:**
    *   `init(self, config)`: Вызывается при инициализации сервисов через `Allocator`. В подклассах здесь создается и настраивается специфичный для типа файла конвейер `ItemIndexer` и сохраняется в `self.attachments["base_indexer"]`.
    *   `get_att(self, key, copy_=False)`: Возвращает объект из словаря `attachments`.

### `File(FileScheme)`

Конкретный подкласс `FileScheme`, представляющий схему **по умолчанию** для файлов, тип которых не распознан или для которых нет специальной схемы.

### `ImageJPG(FileScheme)` и `ImagePNG(FileScheme)`

Конкретные схемы для файлов JPEG и PNG.

*   Устанавливают соответствующие `name` и `mach_patterns`.
*   В методе `init` создают конвейер `ItemIndexer` путем объединения (через оператор `|`) различных модулей индексации изображений:
    *   `Image_MetadataRead` (чтение EXIF/IPTC)
    *   `FaceDetector` (обнаружение лиц)
    *   `ImageTagsFromName` (извлечение тегов из имени файла)
    *   `ImageLLavaDescribe` (генерация описания)
    *   `DeepDunBoru` (тегирование аниме)
*   **Важно:** В текущей реализации некоторые индексаторы явно отключаются (`.enabled = False`), а `DeepDunBoru` включается (`.enabled = True`). Это позволяет гибко настраивать процесс индексации для каждого типа файла.
*   Сконфигурированный `ItemIndexer` сохраняется в `attachments["base_indexer"]`.

### `FileTypeRouter(Service)`

Сервис, отвечающий за выбор правильной `FileScheme` для данного файла.

*   **Назначение:** Предоставить единую точку доступа для получения конфигурации обработки файла.
*   **Атрибуты:**
    *   `file_schemes` (list): Список экземпляров зарегистрированных `FileScheme` (например, `ImageJPG`, `ImagePNG`).
    *   `default` (FileScheme): Экземпляр схемы по умолчанию (`File`).
*   **Методы:**
    *   `get_type_by_path(self, path)`: Получает путь к файлу, извлекает расширение и ищет в `file_schemes` схему, чей `mach_patterns` соответствует расширению (используя `fnmatch`). Возвращает найденную схему или `default`.
    *   `get_type_by_name(self, name)`: Ищет схему по её атрибуту `name` (сравнение без учета регистра). Возвращает найденную схему или `default`.

## Регистрация

Экземпляры `ImageJPG()`, `ImagePNG()` и `FileTypeRouter()` регистрируются в `Allocator.res` при импорте модуля. Это делает их доступными как сервисы во всем приложении.

## Принцип работы

1.  При инициализации приложения `Allocator` вызывает метод `init()` для зарегистрированных схем (`ImageJPG`, `ImagePNG`), которые создают и настраивают свои конвейеры индексации.
2.  Когда системе нужно обработать файл (например, во время индексации), она обращается к сервису `FileTypeRouter`.
3.  `FileTypeRouter.get_type_by_path(file_path)` вызывается для определения соответствующей `FileScheme`.
4.  Из полученной схемы извлекается настроенный конвейер индексации: `scheme.get_att("base_indexer")`.
5.  Этот конвейер используется для обработки файла.

Эта система позволяет легко добавлять поддержку новых типов файлов, просто создав новый подкласс `FileScheme`, определив его `mach_patterns`, настроив конвейер индексации в методе `init` и зарегистрировав экземпляр в `Allocator`.
