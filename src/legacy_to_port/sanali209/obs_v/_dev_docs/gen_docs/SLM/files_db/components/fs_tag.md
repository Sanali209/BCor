# Hierarchical Tag Record (`fs_tag.py`)

Этот модуль определяет класс `TagRecord`, представляющий **иерархическую систему тегов**, хранящуюся в MongoDB. Теги используются для аннотирования и организации записей `FileRecord` (и потенциально других `CollectionRecord`).

## Зависимости

*   `json`, `os`
*   `collections.OrderedDict`
*   `pymongo.UpdateOne`
*   `tqdm`
*   `SLM.files_db.components.File_record_wraper.FileRecord`
*   `SLM.mongoext.wraper.MongoRecordWrapper`, `SLM.mongoext.wraper.FieldPropInfo`

## Класс `TagRecord`

Наследуется от `MongoRecordWrapper`. Предполагается, что для этого класса используется отдельная коллекция MongoDB (например, `tags`).

### Назначение

*   Моделирование иерархической структуры тегов.
*   Управление созданием, поиском, переименованием и удалением тегов.
*   Связывание тегов с записями `FileRecord` (отношение многие-ко-многим).
*   Предоставление утилит для работы с тегами (ремаппинг, отчеты).

### Атрибуты и Поля (`FieldPropInfo`)

*   **`name: str`**: Короткое имя тега (последний компонент `fullName`). Например, `long_hair`.
*   **`fullName: str`**: Полное, уникальное имя тега, отражающее его положение в иерархии. Компоненты разделяются `/` или `\`. Например, `character/female/long_hair`. **Используется как основной идентификатор при связывании с файлами.**
*   **`autotag: bool`**: Флаг, указывающий, был ли тег добавлен автоматически (например, AI-сервисом).
*   **`remap_to_tags: str`**: Строка, содержащая полные имена (`fullName`) других тегов, разделенные точкой с запятой (`;`). Используется для переноса файлов с этого тега на другие теги при вызове `remap_tag()`.
*   **`parent_tag: ObjectId`**: Ссылка (`_id`) на документ родительского тега в MongoDB. Для корневых тегов имеет значение `None`. Определяется автоматически при создании тега на основе `fullName`.

### Иерархия и Связи

*   **Иерархия:** Реализована через поле `parent_tag` и структуру `fullName`. При создании тега (`get_or_create` -> `_get_or_create` -> `create_record_data`) родительский тег определяется из `fullName`, и если он не существует, создается рекурсивно.
*   **Связь с `FileRecord`:** Отношение многие-ко-многим. В документе `FileRecord` есть поле `tags` (список строк), которое хранит `fullName` всех тегов, примененных к этому файлу.

### Ключевые Методы

#### Создание и Поиск

*   **`get_or_create(cls, full_name="", **kwargs)` (classmethod)**: **Основной метод для получения/создания тега.** Ищет тег по `fullName`. Если не найден, создает его (и рекурсивно всех его родителей, если их нет).
*   **`create_record_data(cls, **kwargs)` (classmethod)**: Готовит данные для нового документа тега. Извлекает `name` из `fullName`, находит или создает родительский тег и сохраняет его `_id` в `parent_tag`.
*   **`get_record_by_name(cls, name)` (classmethod)**: Находит тег по `fullName`.
*   **`get_all_tags(cls, root_tags=False)` (classmethod)**: Возвращает список всех тегов, отсортированных по `fullName`. Опция `root_tags=True` возвращает только корневые теги.
*   **`get_tags_of_file(cls, file)` (classmethod)**: Получает список `fullName` тегов из `file.tags` и возвращает список соответствующих объектов `TagRecord`.

#### Навигация по Иерархии и Связям

*   **`parent_tag` (property)**: Возвращает объект `TagRecord` родительского тега. Сеттер позволяет установить родителя по объекту `TagRecord`.
*   **`child_tags(self)`**: Возвращает список прямых дочерних тегов.
*   **`tagged_files(self)`**: Возвращает список объектов `FileRecord`, помеченных этим тегом (ищет файлы, у которых `self.fullName` присутствует в поле `tags`).

#### Модификация Тегов и Связей

*   **`add_to_file_rec(self, file: FileRecord)`**: Добавляет `self.fullName` в список `file.tags` (без дубликатов).
*   **`remove_from_file_rec(self, file: FileRecord)`**: Удаляет `self.fullName` из списка `file.tags`.
*   **`delete(self)`**: **Рекурсивно удаляет тег и всех его потомков.** Перед удалением самого тега, удаляет его `fullName` из списка `tags` у всех связанных файлов с помощью `update_many` и операции `$pull`.
*   **`rename(self, new_name)`**: **Рекурсивно переименовывает тег и всех его потомков.** Обновляет `fullName` и `name`. Если изменился родительский путь, обновляет `parent_tag`. Затем обновляет `fullName` тега в списке `tags` у всех связанных файлов с помощью `bulk_write`.
*   **`remap_tag(self)`**: Переносит файлы с текущего тега на теги, указанные в `remap_to_tags`. Находит все файлы с текущим тегом и добавляет им новые теги.

#### Утилиты

*   **`get_tags_report(cls)` (classmethod)**: Собирает статистику по всем тегам (количество файлов, количество дочерних тегов, флаг `autotag`), сохраняет ее в JSON-файл (`tags_report.json`) и пытается открыть этот файл.

## Использование

`TagRecord` используется для создания и управления словарем тегов. Метод `get_or_create` является предпочтительным способом получения экземпляров тегов. Методы `add_to_file_rec` и `remove_from_file_rec` используются для связи тегов с файлами. Методы `delete` и `rename` обеспечивают целостность данных при модификации иерархии тегов.
