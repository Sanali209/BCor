# DeepDanbooru Image Tagger Indexer (`deepdanboru.py`)

Этот модуль реализует `ItemFieldIndexer` для автоматического тегирования изображений с использованием модели DeepDanbooru. Он извлекает теги, связанные с содержимым изображения (особенно полезно для аниме-стиля), и добавляет их в запись файла.

## Зависимости

*   `SLM.files_db.components.fs_tag.TagRecord`: Для управления записями тегов в базе данных (получение или создание).
*   `SLM.files_db.indexers.image.content_md5.files_db_indexer`: Базовый класс для индексаторов полей изображений.
*   `SLM.indexerpyiplain.idexpyiplain.ItemIndexer`: Базовый класс для родительского индексатора (используется для type hinting и доступа к `shared_data`).
*   `SLM.vision.imagetotext.backends.deep_dan_boru.DeepDanbury`: Бэкенд-модель, выполняющая фактическое распознавание тегов на изображении.
*   `SLM.files_db.components.File_record_wraper.FileRecord`: Для доступа к полному пути файла.

## Класс `DeepDunBoru`

```python
class DeepDunBoru(files_db_indexer):
    # ... реализация ...
```

*   **Наследование:** Наследует от `files_db_indexer` (который является `ItemFieldIndexer`).
*   **`fieldName`:** Установлен в `"deepdanbooru"`. Используется для отметки записей как обработанных этим индексатором.
*   **Метод `index(self, parent_indexer: ItemIndexer, item, need_index)`:**
    *   **Назначение:** Выполняет процесс тегирования изображения с помощью DeepDanbooru.
    *   **Процесс:**
        1.  Получает объект `FileRecord` для доступа к пути файла.
        2.  Создает экземпляр бэкенда `DeepDanbury`. *(Примечание: Создание экземпляра здесь может быть неоптимальным для производительности).*
        3.  Вызывает `DeepDanbury.get_label_from_path()` для получения строки тегов.
        4.  Обрабатывает полученные теги: разделяет строку, удаляет пробелы.
        5.  Для каждого тега:
            *   Добавляет префикс `"auto/deepdanboru/"` для указания источника тега и его автоматического происхождения.
            *   Использует `TagRecord.get_or_create()` для обеспечения наличия тега в базе данных.
            *   Добавляет полное имя тега (с префиксом) к списку тегов (`item['tags']`).
        6.  Удаляет дубликаты из обновленного списка тегов.
        7.  Обновляет поле `item['tags']`.
        8.  Устанавливает флаг `item_indexed = True` у родительского индексатора.
        9.  Вызывает `self.mark_as_indexed()` для добавления `"deepdanbooru"` в поле `item['indexed_by']`.

## Интеграция

Этот индексатор предназначен для использования в цепочке индексаторов, управляемой родительским `ItemIndexer` (вероятно, `ImageIndexer`). Он вызывается во время процесса индексации изображений, если он еще не был применен к данной записи (проверяется с помощью `isNeedIndex`, унаследованного от `files_db_indexer`).
