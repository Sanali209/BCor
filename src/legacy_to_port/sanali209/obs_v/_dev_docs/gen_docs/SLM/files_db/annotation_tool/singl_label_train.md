# Single-Label Image Classification Training (`singl_label_train.py`)

Этот скрипт предназначен для обучения (fine-tuning) модели классификации изображений с одной меткой на основе аннотаций, хранящихся в SLM. Он использует библиотеки Hugging Face `transformers` для загрузки предобученной модели и `feature_extractor`, PyTorch для самого процесса обучения, и интегрируется с компонентами SLM (`AnnotationJob`, `AnnotationRecord`, `ImageThumbCache`) для получения данных.

## Зависимости

*   **Основные ML/DL:** `torch`, `transformers`, `sklearn` (для `train_test_split`), `PIL (Pillow)`.
*   **Утилиты:** `os`, `random`, `tqdm`.
*   **Hugging Face Hub:** `huggingface_hub` (для опциональной загрузки модели).
*   **SLM:** `SLM.appGlue.core.Allocator`, `SLM.files_data_cache.thumbnail.ImageThumbCache`, `SLM.files_db.annotation_tool.annotation` (`AnnotationRecord`, `AnnotationJob`).

## Основные компоненты

1.  **`ImageClassificationDataset(Dataset)`**:
    *   Кастомный класс датасета PyTorch.
    *   Принимает список записей (кортежи `(путь_к_миниатюре, метка)`), `feature_extractor` и маппинг `label2id`.
    *   В `__getitem__`:
        *   Загружает изображение по пути (миниатюру).
        *   Применяет `feature_extractor` для предобработки изображения.
        *   Преобразует строковую метку в числовой ID.
        *   Возвращает словарь `{"pixel_values": ..., "labels": ...}`.
    *   Включает базовую обработку ошибок при загрузке/препроцессинге изображения.

2.  **`evaluate_model(model, dataloader, criterion, device)`**:
    *   Функция для оценки производительности модели на заданном даталоадере (обычно валидационном).
    *   Переводит модель в режим `eval()`.
    *   Итерирует по батчам без вычисления градиентов (`torch.no_grad()`).
    *   Вычисляет среднюю потерю (loss) и точность (accuracy).
    *   Возвращает `avg_loss`, `accuracy`.

3.  **`train_model(...)`**:
    *   Основная функция, реализующая цикл обучения модели.
    *   Принимает модель, даталоадеры (train/val), оптимизатор, функцию потерь, устройство, количество эпох, пути сохранения, информацию о модели/экстракторе и опциональный ID репозитория Hugging Face Hub.
    *   **Цикл по эпохам:**
        *   Переводит модель в режим `train()`.
        *   **Цикл по обучающим батчам:**
            *   Перемещает данные на `device`.
            *   Обнуляет градиенты оптимизатора.
            *   Выполняет прямой проход (forward pass).
            *   Вычисляет потери.
            *   Выполняет обратный проход (backward pass).
            *   Делает шаг оптимизатора.
            *   Собирает статистику по потерям и точности для эпохи.
        *   Вызывает `evaluate_model` для оценки на валидационной выборке.
        *   Печатает результаты эпохи (train/val loss & accuracy).
        *   **Сохранение лучшей модели:** Если текущая точность на валидации (`val_acc`) лучше предыдущей лучшей, сохраняет модель и `feature_extractor` локально в формате, совместимом с Hugging Face (`save_pretrained`).
    *   **Загрузка на Hugging Face Hub:** Если указан `hub_repo_id`, после обучения пытается загрузить лучшую сохраненную модель в указанный репозиторий. Использует токен (рекомендуется устанавливать через переменную окружения `HUGGING_FACE_TOKEN_DOCS`).
    *   Возвращает путь к директории с лучшей сохраненной моделью.

4.  **`predict_image(...)`**:
    *   Функция для выполнения инференса (предсказания) на одном изображении.
    *   Загружает изображение, применяет `feature_extractor`, пропускает через модель.
    *   Возвращает предсказанную метку (строку) и уверенность модели (probability).
    *   *(Примечание: Эта функция не используется в основном потоке выполнения скрипта `if __name__ == "__main__":`)*.

## Основной блок (`if __name__ == "__main__":`)

Этот блок оркестрирует весь процесс обучения:

1.  **Инициализация SLM:** Настраивает пути и имя базы данных, инициализирует модули SLM через `Allocator`.
2.  **Конфигурация:** Задает параметры обучения:
    *   `MODEL_NAME`: Имя базовой модели из Hugging Face Hub (`google/vit-base-patch16-224`).
    *   `NUM_EPOCHS`: Количество эпох (7).
    *   `BATCH_SIZE`: Размер батча (16).
    *   `LEARNING_RATE`: Скорость обучения (5.646e-05).
    *   `SAVE_DIR`: Папка для сохранения результатов (`training_output`).
    *   `TEST_SPLIT_SIZE`: Доля данных для валидации (0.2).
    *   `RANDOM_SEED`: Для воспроизводимости (42).
    *   `HUB_MODEL_ID`: ID репозитория на Hugging Face Hub для загрузки (`sanali209/nsfwfilter`).
3.  **Установка Seed:** Устанавливает random seed для Python, PyTorch и CUDA.
4.  **Выбор устройства:** Определяет `device` (CUDA, если доступно, иначе CPU).
5.  **Подготовка данных:**
    *   Загружает `AnnotationJob` по имени ("NSFWFilter").
    *   Получает все аннотированные записи (`get_all_annotated`).
    *   Извлекает уникальные метки из `job.choices`.
    *   Создает словари `label2id` и `id2label`.
    *   Формирует список `records`, содержащий кортежи `(путь_к_миниатюре, метка)`. Использует `ImageThumbCache` для получения пути к миниатюре среднего размера ("medium"). Пропускает файлы с неподдерживаемыми расширениями.
    *   Разделяет `records` на обучающую (`train_annotations`) и валидационную (`val_annotations`) выборки с использованием `train_test_split` и стратификации по меткам.
6.  **Загрузка Feature Extractor:** Загружает `AutoFeatureExtractor` для `MODEL_NAME`.
7.  **Создание Dataset/DataLoader:** Создает экземпляры `ImageClassificationDataset` и `DataLoader` для обучающей и валидационной выборок. *(Примечание: `num_workers` установлен в 0 из-за бага, упомянутого в комментарии)*.
8.  **Загрузка Модели:** Загружает `AutoModelForImageClassification` для `MODEL_NAME`, указывая количество меток и маппинги `label2id`/`id2label`. `ignore_mismatched_sizes=True` позволяет адаптировать классификационную голову модели к новому числу классов.
9.  **Настройка Оптимизатора и Потерь:** Создает оптимизатор `AdamW` и функцию потерь `CrossEntropyLoss`.
10. **Запуск Обучения:** Вызывает функцию `train_model` со всеми подготовленными компонентами и параметрами.

## Интеграция с SLM

*   Скрипт напрямую использует `AnnotationJob.get_by_name()` и `AnnotationJob.get_all_annotated()` для получения данных для обучения.
*   Он полагается на `AnnotationRecord.file` для доступа к `FileRecord` и его `full_path`.
*   Использует `ImageThumbCache.instance().get_thumb()` для получения путей к кэшированным миниатюрам, что ускоряет загрузку данных по сравнению с загрузкой оригинальных изображений.
*   Требует инициализации `Allocator` и конфигурации путей (`fileDataManager.path`, `mongoConfig.database_name`).

Этот скрипт представляет собой пример того, как аннотации, собранные в SLM, могут быть использованы для обучения модели классификации изображений с использованием стандартных инструментов ML/DL.
