# Catalogs (`catalogs/`)

Этот подпакет содержит реализацию иерархической системы "каталогов" для организации записей `FileRecord` в базе данных SLM.

## Структура

*   [`__init__.py`](__init__.md): Пустой файл, необходимый для распознавания директории как пакета Python.
*   [`catalog.py`](catalog.md): Определяет класс `CatalogRecord`, который моделирует иерархический каталог и предоставляет методы для управления каталогами и их связями с файлами.

## Назначение

Система каталогов предоставляет альтернативный или дополнительный способ иерархической классификации файлов по сравнению с системой тегов (`fs_tag.py`). Каталоги могут представлять более стабильную структуру, например, соответствующую физической организации папок или предопределенным категориям контента.

## Ключевые особенности

*   **Иерархия:** Каталоги организованы в дерево с помощью поля `fullName` (путь с разделителями) и ссылки `parentCatalog` на родительский документ.
*   **Связь с файлами:** Каталоги связаны с `FileRecord` через поле `catalogs` (список строк `fullName`) в документах файлов.
*   **Управление:** Предоставляются методы для создания (`get_or_create`), поиска, переименования (`rename`) и удаления (`delete`) каталогов с автоматическим обновлением иерархии и ссылок в связанных файлах.
*   **Инициализация:** Коллекция `"catalogs_records"` и индекс по `fullName` создаются при запуске приложения через функцию `init`, зарегистрированную в `allocator.Allocator`.

## Сравнение с Тегами (`TagRecord`)

Реализация `CatalogRecord` очень похожа на `TagRecord`, но:
*   Использует другие имена полей (`parentCatalog`, `catalogs`).
*   Не имеет полей `autotag` и `remap_to_tags`.
*   Хранится в отдельной коллекции (`"catalogs_records"`).
