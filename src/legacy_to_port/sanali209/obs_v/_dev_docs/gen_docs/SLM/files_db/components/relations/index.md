# Relations (`relations/`)

Этот подпакет отвечает за управление и использование **связей (отношений)** между различными записями (`CollectionRecord`) в базе данных SLM, а также за реализацию **поиска схожести** на основе векторных представлений.

## Структура

### Файлы

*   [`__init__.py`](__init__.md): Пустой файл, необходимый для распознавания директории как пакета Python.
*   [`relation.py`](relation.md): Определяет класс `RelationRecord`, который является основной моделью данных для хранения направленных связей между записями в MongoDB.
*   [`dubsearch.py`](dubsearch.md): Реализует логику поиска визуальной схожести (дубликатов) между файлами (`FileRecord`) и объектами (`Detection`) с использованием `VectorDB` и различных CNN-энкодеров. Управляет созданием связей `RelationRecord` типа `"similar_search"` и `"similar_obj_search"`.

### Поддиректории

*   `embeddings/`: (Предположительно) Содержит код, связанный с генерацией или управлением векторными представлениями (эмбеддингами), используемыми для поиска схожести. *(Требуется дальнейшее документирование)*
*   `pats_md5/`: (Предположительно) Содержит код, связанный с использованием MD5-хэшей файлов, возможно, для предварительной фильтрации или идентификации точных дубликатов. *(Требуется дальнейшее документирование)*

## Назначение

Пакет `relations` предоставляет механизмы для:
1.  **Моделирования связей:** Определения и хранения различных типов направленных отношений между элементами данных (файлами, объектами и т.д.) с помощью `RelationRecord`.
2.  **Поиска схожести:** Использования векторных представлений и `VectorDB` для нахождения визуально похожих файлов и объектов (`dubsearch.py`).
3.  **Построения графа связей:** Создания и управления графом схожести и других типов отношений в базе данных.

Этот пакет является ключевым для функций обнаружения дубликатов, поиска похожих изображений и потенциально других возможностей, основанных на связях между данными.
