# Image Indexers (`indexers/image/`)

Этот пакет содержит набор индексаторов для извлечения метаданных, тегов и информации из изображений. Каждый индексатор специализируется на определенном аспекте анализа и обработки изображений.

## Базовый функционал

*   [`content_md5.py`](content_md5.md): Базовый класс `files_db_indexer`, предоставляющий общую функциональность для отслеживания состояния индексации.

## Индексаторы извлечения информации

*   [`metadata_read.py`](metadata_read.md): Извлекает метаданные из EXIF, IPTC, XMP и других источников, нормализует их и сохраняет в стандартизированном формате.

*   [`face_detection.py`](face_detection.md): Обнаруживает лица на изображениях с помощью различных бэкендов детекции, сохраняет вырезанные изображения лиц и создает соответствующие записи в базе данных.

*   [`llava_describe.py`](llava_describe.md): Использует модель LLaVA (Large Language and Vision Assistant) для создания подробных текстовых описаний изображений.

## Индексаторы тегирования

*   [`deepdanboru.py`](deepdanboru.md): Использует модель DeepDanbooru для автоматического тегирования изображений, особенно эффективен для аниме и иллюстраций.

*   [`tags_from_name.py`](tags_from_name.md): Извлекает теги из имени файла с использованием NLP-конвейера для токенизации и нормализации.

## Использование индексаторов

Индексаторы могут использоваться:
1.  По отдельности, когда нужен конкретный тип анализа.
2.  В составе цепочки индексации, когда нужно извлечь разные типы информации.

Типичная последовательность индексации:
1.  Чтение метаданных (`metadata_read`)
2.  Извлечение тегов из имени (`tags_from_name`)
3.  Анализ содержимого:
    *   Поиск лиц (`face_detection`)
    *   Анализ стиля и содержимого (`deepdanboru`)
    *   Генерация описания (`llava_describe`)

## Общие характеристики

1.  **Наследование:**
    *   Все индексаторы наследуются от `files_db_indexer`.
    *   Каждый реализует метод `index()` для своей специфической функциональности.
    *   Каждый использует систему отслеживания состояния индексации через `indexed_by`.

2.  **Интеграция с базой данных:**
    *   Работают с `FileRecord` для доступа к файлам.
    *   Используют `TagRecord` для управления тегами.
    *   Поддерживают общее хранилище метаданных.

3.  **Расширяемость:**
    *   Можно добавлять новые индексаторы для других аспектов анализа.
    *   Можно модифицировать существующие для поддержки новых моделей или источников данных.
    *   Поддерживается композиция индексаторов через систему цепочек.
