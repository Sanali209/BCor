# Face Detection Indexer (`face_detection.py`)

Этот модуль реализует индексатор для обнаружения лиц на изображениях. Он использует несколько бэкендов детекции объектов, применяет Non-Maximum Suppression (NMS) для фильтрации перекрывающихся обнаружений, сохраняет вырезанные изображения лиц и создает соответствующие записи в базе данных.

## Зависимости

*   **Системные:**
    *   `os`: Для работы с путями файлов
    *   `cv2`: OpenCV для NMS и обработки изображений

*   **SLM Core:**
    *   `SLM.Allocator`: Для получения доступных детекторов объектов
    *   `SLM.appGlue.DesignPaterns.MessageSystem`: Для подписки на изменения конфигурации
    *   `SLM.appGlue.helpers.ImageHelper`: Для вычисления MD5 изображений

*   **Object Recognition:**
    *   `SLM.files_db.object_recognition.object_recognition`:
        *   `DetectionObjectClass`: Для категоризации обнаруженных объектов
        *   `Detection`: Для хранения данных об обнаружениях
        *   `Recognized_object`: Для связывания обнаружений с распознанными объектами

*   **Base Classes:**
    *   `SLM.indexerpyiplain.idexpyiplain.ItemFieldIndexer, ItemIndexer`: Базовые классы для индексаторов

*   **Vision:**
    *   `SLM.vision.objectdetectors`: Модуль детекторов объектов
    *   `SLM.vision.objectdetectors.object_detect.ObjectDetectorProvider`: Интерфейс для детекторов

### Инициализация

*   **`__init__`:**
    *   Устанавливает `fieldName = "face_detection"`.
    *   Подписывается на изменения конфигурации через `MessageSystem`.
    *   Задает путь для сохранения вырезанных лиц (`path`). *TODO: перенести в конфигурацию*.

*   **`on_config_change(self, section, attribute, value)`:**
    *   Обрабатывает изменения конфигурации.
    *   Если `section == "Indexer"` и `attribute == "detectFaces"`, обновляет `self.enabled`.

### Основная функциональность

**`index(self, parent_indexer: ItemIndexer, item, need_index)`:**

1.  **Подготовка:**
    *   Получает `FileRecord` для доступа к файлу изображения.
    *   Получает список доступных бэкендов детекции через `ObjectDetectorProvider`.

2.  **Детекция:**
    *   Для каждого бэкенда (если он еще не применялся к этому файлу):
        *   Запускает детекцию через `backend.detect_by_path()`.
        *   Фильтрует результаты по размеру (игнорирует детекции меньше 20x20).
        *   Добавляет имя бэкенда к результатам.
        *   Отмечает бэкенд как использованный в `item["backend_indexed"]`.

3.  **Фильтрация результатов:**
    *   Применяет OpenCV NMS для удаления перекрывающихся детекций:
        *   Использует порог уверенности 0.5.
        *   Использует порог IoU 0.4.

4.  **Обработка результатов:**
    *   Для каждого отфильтрованного результата:
        1.  **Сохранение изображения:**
            *   Вычисляет MD5-хеш вырезанного изображения лица.
            *   Создает путь с шардингом (первые 2 символа хеша).
            *   Конвертирует в RGB если нужно.
            *   Сохраняет как JPEG с максимальным качеством.

        2.  **Создание записей в БД:**
            *   Создает/получает `DetectionObjectClass` для типа "лицо".
            *   Создает запись `Detection` с данными об обнаружении (регион, уверенность, путь к сохраненному изображению).
            *   Связывает с родительским изображением через `parent_image_id`.
            *   Создает и связывает `Recognized_object` с именем "unknown".

        3.  **Тегирование:**
            *   Добавляет тег "object_detect/face" к файлу.

## Использование

Этот индексатор является частью цепочки индексации изображений и отвечает за:
1.  Обнаружение лиц на изображениях с помощью различных бэкендов детекции.
2.  Сохранение вырезанных изображений лиц для дальнейшего использования.
3.  Создание структуры данных в БД для связывания обнаружений с исходными изображениями.
4.  Тегирование изображений, содержащих лица.

Это позволяет в дальнейшем выполнять поиск изображений по наличию лиц, анализировать и группировать лица, и т.д.
