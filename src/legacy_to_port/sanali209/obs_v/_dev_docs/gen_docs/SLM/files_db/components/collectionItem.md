# Base Collection Item (`collectionItem.py`)

Этот модуль определяет класс `CollectionRecord`, который служит **абстрактным базовым классом** для всех типов записей (документов), хранящихся в основной коллекции MongoDB базы данных SLM (вероятно, коллекции `files`). Он использует `MongoRecordWrapper` для предоставления объектно-ориентированного интерфейса к документам MongoDB и реализует механизм фабрики для работы с различными типами записей.

## Зависимости

*   `typing.Dict`
*   `SLM.appGlue.core.Event`
*   `SLM.mongoext.wraper.MongoRecordWrapper`, `SLM.mongoext.wraper.FieldPropInfo`

## Класс `CollectionRecord`

Наследуется от `MongoRecordWrapper`.

### Назначение

*   Определить **общую структуру и поля**, присущие большинству (или всем) типам записей в коллекции.
*   Реализовать **механизм фабрики** для создания экземпляров конкретных подклассов-оберток на основе типа записи, хранящегося в документе MongoDB.
*   Предоставить **базовые методы поиска**, которые автоматически фильтруют результаты по типу записи.

### Атрибуты класса

*   **`itemTypeMap: Dict[str, type]`**: Статический словарь, действующий как **реестр типов**. Он сопоставляет строковый идентификатор типа записи (значение поля `item_type` в MongoDB) с соответствующим классом Python (наследником `CollectionRecord`). Этот словарь заполняется при импорте конкретных подклассов (например, `FileRecord`, `WebLinkRecord`).
*   **`onDelete: Event`**: Экземпляр `Event` из `SLM.appGlue.core`. Позволяет подписчикам реагировать на событие удаления экземпляра `CollectionRecord`.

### Определения полей (`FieldPropInfo`)

Класс декларирует ряд общих полей с помощью `FieldPropInfo`, предоставляя имя поля в MongoDB, тип Python и значение по умолчанию:

*   `itemType: str` (По умолчанию: `'CollectionRecord'`) - **Ключевое поле**. Хранит строковый идентификатор типа записи. **Важно:** Подклассы должны переопределять значение по умолчанию для этого поля, чтобы методы `find`/`find_one` работали корректно.
*   `favorite: bool` (По умолчанию: `False`)
*   `hidden: bool` (По умолчанию: `False`)
*   `rating: int` (По умолчанию: `0`)
*   `document_content: str` (По умолчанию: `''`) - Вероятно, для хранения извлеченного текстового содержимого.
*   `full_text_search: str` (По умолчанию: `''`) - Поле, предназначенное для индексации полнотекстовым поиском MongoDB.
*   `title: str` (По умолчанию: `''`)
*   `description: str` (По умолчанию: `''`)
*   `notes: str` (По умолчанию: `''`)
*   `ai_expertise: list` (По умолчанию: `[]`) - Вероятно, для хранения результатов анализа ИИ (тип элементов не указан строго, но используется `str` в `FieldPropInfo`).
*   `file_content_md5: str` (По умолчанию: `''`) - MD5 хэш содержимого файла (если применимо).
*   `metadata_dirty: bool` (По умолчанию: `False`) - Флаг, указывающий на необходимость обновления метаданных.

### Статические методы (Фабрика и Поиск)

*   **`get_record_wrapper(cls, record_id)`**:
    *   Находит документ MongoDB по `record_id`.
    *   Читает значение поля `item_type` из документа.
    *   Использует `itemTypeMap` для получения соответствующего класса Python.
    *   Возвращает экземпляр этого класса, инициализированный `record_id`.
*   **`get_record_wrapper2(cls, record_id, record_type)`**:
    *   Более производительная версия `get_record_wrapper`.
    *   Принимает `record_type` (строку) в качестве аргумента, избегая необходимости запроса к БД для определения типа.
    *   Напрямую использует `itemTypeMap` для получения класса и создания экземпляра.
*   **`find_one(cls, query)`**:
    *   Переопределяет базовый метод.
    *   Автоматически добавляет к `query` условие фильтрации по полю `item_type`. Значение для фильтра берется из **значения по умолчанию** поля `itemType` **вызывающего класса** (например, если вызвать `FileRecord.find_one(...)`, будет добавлен фильтр `{'item_type': 'FileRecord'}`).
    *   Выполняет `collection().find_one(query)`.
    *   Возвращает экземпляр **вызывающего класса** или `None`.
*   **`find(cls, query, sort_query=None)`**:
    *   Аналогично `find_one`, автоматически добавляет фильтр по `item_type` к `query`.
    *   Выполняет `collection().find(query)` (с возможной сортировкой `sort_query`).
    *   Возвращает список экземпляров **вызывающего класса**.

### Методы экземпляра

*   **`get_thumb(self, size=None)`**:
    *   Метод-заглушка (placeholder).
    *   Предназначен для переопределения в подклассах, которые могут предоставлять миниатюры (например, `FileRecord`). Должен возвращать путь к файлу миниатюры или объект изображения.

### Использование

`CollectionRecord` не предназначен для прямого инстанцирования (кроме как внутри методов поиска базового класса `MongoRecordWrapper`). Вместо этого используются его подклассы (например, `FileRecord`), которые регистрируются в `itemTypeMap`. Методы `get_record_wrapper` или `get_record_wrapper2` используются для получения объекта правильного типа при работе с записями, тип которых заранее неизвестен или может варьироваться. Методы `find`/`find_one`, вызванные у конкретного подкласса, обеспечивают удобный поиск записей именно этого типа.
