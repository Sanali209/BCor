# Folder Indexing (`index_folder.py`)

Этот модуль содержит функции для запуска процесса индексации файлов, представленных в базе данных SLM как `FileRecord`. Индексация включает применение специфичных для типа файла обработчиков ("индексаторов") для извлечения метаданных, генерации превью, вычисления эмбеддингов и т.д.

## Зависимости

*   `os`: Для проверки существования файлов и установки переменных окружения.
*   `concurrent.futures.ThreadPoolExecutor`: Для параллельного выполнения индексации.
*   `loguru`: Для логирования.
*   `tqdm`: Для отображения прогресса.
*   `SLM.files_db.components.CollectionRecordScheme.FileTypeRouter`: Для получения нужного индексатора по типу файла.
*   `SLM.files_db.components.File_record_wraper.FileRecord`, `get_file_record_by_folder`: Для работы с записями файлов.
*   `SLM.iterable.bach_builder.BatchBuilder`: Для разделения файлов на пакеты.

## Настройка окружения

В начале скрипта устанавливаются переменные окружения (`DATA_CACHE_MANAGER_PATH`, `APPDATA`, `MONGODB_NAME`) с **жестко закодированными путями**. Это делает модуль зависимым от конкретной конфигурации и затрудняет его использование как чистой библиотеки. Вероятно, он предназначен для запуска в определенном окружении или как отдельный скрипт.

```python
def index_folder_one_thread(path: str):
    # ... реализация ...
```

*   **Назначение:** Индексирует файлы в указанной папке `path` и ее подпапках **последовательно**.
*   **Процесс:**
    1.  Находит или создает `FileRecord` для файлов в `path` с помощью `get_file_record_by_folder`.
    2.  Для каждого файла определяет нужный "базовый индексатор" через `FileTypeRouter`.
    3.  Проверяет наличие файла на диске.
    4.  Вызывает метод `index()` соответствующего индексатора, который модифицирует данные записи.
    5.  Если индексация произошла, обновляет запись в MongoDB (неэффективно используя `update_bulk` для одного элемента).

## Функция `index_folder`

```python
def index_folder(query=None, worckers=4):
    # ... реализация ...
```

*   **Назначение:** Индексирует `FileRecord`, соответствующие MongoDB-запросу `query`, используя **несколько рабочих потоков**.
*   **Параметры:**
    *   `query` (dict, optional): Фильтр MongoDB для выбора `FileRecord`. По умолчанию `{}` (все).
    *   `worckers` (int, optional): Количество потоков для параллельной обработки. По умолчанию 4.
*   **Процесс:**
    1.  Находит `FileRecord` по запросу `query`.
    2.  Разбивает найденные записи на пакеты с помощью `BatchBuilder`.
    3.  Для каждого пакета:
        *   Запускает индексацию файлов параллельно с помощью `ThreadPoolExecutor`.
        *   Процесс индексации одного файла аналогичен `index_folder_one_thread` (получение индексатора, проверка файла, вызов `indexer.index()`).
        *   Собирает измененные данные записей.
        *   Обновляет все измененные записи пакета в MongoDB одним вызовом `FileRecord.update_bulk()`.

## Использование как скрипта (`if __name__ == '__main__':`)

*   Модуль можно запустить напрямую (`python index_folder.py`).
*   При запуске настраивается логирование в `log.log`.
*   Вызывается `index_folder` с жестко закодированным путем, что инициирует многопоточную индексацию файлов в этой папке.

## Общее назначение

Модуль предоставляет ключевую функциональность для обработки и обогащения данных о файлах в базе SLM. Он координирует применение различных индексаторов к файлам в зависимости от их типа, позволяя извлекать метаданные, генерировать эмбеддинги и выполнять другие задачи по обработке файлов.
