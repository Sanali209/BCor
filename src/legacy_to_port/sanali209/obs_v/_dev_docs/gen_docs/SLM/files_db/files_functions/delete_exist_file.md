# Delete Non-Existing File Records (`delete_exist_file.py`)

Этот модуль предоставляет функцию для очистки базы данных MongoDB от записей `FileRecord`, которые ссылаются на файлы, более не существующие в файловой системе.

## Зависимости

*   `os`: Для проверки существования файлов (`os.path.exists`).
*   `tqdm`: Для отображения индикатора прогресса при переборе записей.
*   `SLM.files_db.components.File_record_wraper.FileRecord`: Класс-обертка для работы с записями файлов.
*   `SLM.mongoext.MongoClientEXT_f.MongoClientExt`: Класс для взаимодействия с MongoDB (используется в аннотации типа).

*   **Назначение:** Синхронизировать коллекцию `FileRecord` с файловой системой путем удаления записей о файлах, которые были удалены или перемещены.
*   **Параметры:**
    *   `db_client: MongoClientExt`: Экземпляр клиента MongoDB (хотя в текущей реализации он явно не используется).
*   **Процесс:**
    1.  Извлекает все документы из коллекции, связанной с `FileRecord`.
    2.  Итерирует по всем найденным записям, отображая прогресс с помощью `tqdm`.
    3.  Для каждой записи:
        *   Создает объект `FileRecord`.
        *   Получает полный путь к файлу (`file_rec.full_path`).
        *   Проверяет существование файла по этому пути (`os.path.exists(path)`).
        *   Если файл **не найден**, выводит сообщение об удалении и вызывает `file_rec.delete_rec()` для удаления записи из MongoDB.
*   **Возвращаемое значение:** Нет (функция выполняет операции напрямую).

## Использование

Эта функция полезна для поддержания целостности базы данных, особенно если файлы могут удаляться или перемещаться вне приложения SLM. Ее запуск позволяет удалить "осиротевшие" записи, которые больше не соответствуют реальным файлам.
