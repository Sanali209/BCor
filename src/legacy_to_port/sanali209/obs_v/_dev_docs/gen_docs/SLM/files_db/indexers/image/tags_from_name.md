# Tags From Filename Indexer (`tags_from_name.py`)

Этот модуль реализует индексатор для извлечения тегов из имени файла путем его токенизации и нормализации с помощью NLP-конвейера.

## Зависимости

*   **NLP компоненты** (`SLM.NLPSimple.NLPPipline`):
    *   `NLPTextReplace`: Замена символов
    *   `NLPTextTokenize`: Токенизация текста
    *   `NLPTokensToLoverCase`: Приведение к нижнему регистру
    *   `NLPTokensDeleteStopWords`: Удаление стоп-слов
    *   `NLPTokensDeleteShort`: Удаление коротких токенов
    *   `NLPTokensDeleteDuplicates`: Удаление дубликатов
    *   `NLPTokensStripSpacesOperation`: Удаление пробелов
    *   `NLPTokensDeleteIntegers`: Удаление чисел
    *   `NLPTokensDeleteHexadecimal`: Удаление шестнадцатеричных чисел
    *   `NLPTokensDeleteRandString`: Удаление случайных строк
    *   `NLPTokensSetBagOfWords`, `NLPTokensDeleteNotInBagOfWords`: Работа со словарем допустимых слов
    *   `load_bag_of_words`: Загрузка словаря

*   **Базовые компоненты:**
    *   `SLM.files_db.indexers.image.content_md5.files_db_indexer`
    *   `SLM.files_db.components.fs_tag.TagRecord`
    *   `SLM.indexerpyiplain.idexpyiplain.ItemIndexer`

### Конфигурация

*   **`bag_of_words_load_path`**: Путь к JSON-файлу со словарем допустимых слов (`"D:\data\bags_of_words.json"`).
*   **`bag_of_words_data`**: Загруженные данные словаря (кэшируются на уровне класса).
*   **`fieldName`**: Установлен как `"Tags_from_name"`.

### Основная функциональность

**`index(self, parent_indexer: ItemIndexer, item, need_index)`:**

1.  **Подготовка:**
    *   Получает `FileRecord` для доступа к полному пути файла.
    *   Создает экземпляр `NLPPipline`.
    *   Устанавливает путь к файлу как входной текст.
    *   Загружает словари допустимых слов.

2.  **Настройка NLP-конвейера:**
    
    Добавляет операции в следующем порядке:
    
    a) **Предварительная обработка текста:**
       *   Замена специальных символов на пробелы:
           *   `.`, `_`, `-`
           *   `[`, `]`, `(`, `)`
           *   `+`, `=`, `\`
    
    b) **Токенизация и нормализация:**
       *   Разбивка на токены
       *   Приведение к нижнему регистру
       *   Удаление пробелов
    
    c) **Фильтрация токенов:**
       *   Удаление стоп-слов
       *   Удаление чисел
       *   Удаление шестнадцатеричных чисел
       *   Удаление токенов короче 3 символов
       *   Удаление дубликатов
       *   Удаление случайных строк
       *   Удаление слов, отсутствующих в словаре

3.  **Обработка результатов:**
    *   Запускает NLP-конвейер.
    *   Получает текущий список тегов файла.
    *   Для каждого полученного токена:
        *   Добавляет префикс `"from_name/"`.
        *   Создает или получает `TagRecord`.
        *   Добавляет полное имя тега в список.
    *   Удаляет дубликаты из списка тегов.
    *   Обновляет поле `tags` в записи файла.
    *   Устанавливает флаги индексации.

## Отличительные особенности

1.  **Использование NLP:**
    *   Комплексная обработка текста с помощью конвейера операций.
    *   Гибкая система фильтрации и нормализации токенов.
    *   Использование словаря допустимых слов для контроля качества тегов.

2.  **Особенности тегирования:**
    *   Все теги получают префикс `"from_name/"` для указания их источника.
    *   Применяется строгая фильтрация (длина >= 3, только слова из словаря).
    *   Автоматическое удаление дубликатов.

3.  **Предварительная обработка:**
    *   Тщательная очистка от специальных символов.
    *   Удаление чисел и случайных строк.
    *   Работа с распространенными форматами именования файлов.
