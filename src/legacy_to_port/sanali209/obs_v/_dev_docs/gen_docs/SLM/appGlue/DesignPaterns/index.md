# Design Patterns (`DesignPaterns/`)

**Примечание:** В названии директории, вероятно, есть опечатка ("Paterns" вместо "Patterns").

Этот подмодуль `appGlue` содержит реализации или базовые классы для некоторых распространенных паттернов проектирования. Эти паттерны помогают решать типичные проблемы проектирования и способствуют созданию более гибкого, переиспользуемого и поддерживаемого кода.

## Реализованные паттерны

*   **Строитель (Builder):**
    *   [`AbstractBuilder.py`](AbstractBuilder.md): Предоставляет простой абстрактный базовый класс `Builder` для пошагового конструирования сложных объектов.
*   **Фабрика (Factory):**
    *   [`factory.py`](factory.md): Содержит базовые классы `Factory` и `StaticFactory`, а также `StaticFactoryRepo` для управления экземплярами фабрик. Используется для инкапсуляции логики создания объектов.
*   **Издатель-Подписчик / Шина событий (Publish-Subscribe / Event Bus):**
    *   [`MessageSystem.py`](MessageSystem.md): Реализует простую статическую систему обмена сообщениями для слабой связи между компонентами.
*   **Спецификация (Specification):**
    *   [`specification.py`](specification.md): Предоставляет классы для инкапсуляции бизнес-правил или критериев выбора (`Specification`, `AndSpecification`, `OrSpecification`, `NotSpecification`). Включает также парсер (`SpecificationQueryParser`) и строитель (`SpecificationBuilder`) для создания спецификаций.

## Другие файлы

*   [`backend.py`](backend.md): В настоящее время пустой файл.
*   `__init__.py`: Пустой файл, обозначающий пакет Python.

Эти паттерны используются в других частях `appGlue` (например, `DAL`, `binding`) для организации кода и управления зависимостями.
