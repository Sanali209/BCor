# Data Access Layer Core (`DAL.py`)

Этот файл определяет основные классы и концепции для слоя доступа к данным (Data Access Layer - DAL) в рамках `appGlue`. Он включает механизмы для описания схем данных, преобразования данных, маппинга и абстрактный интерфейс `DataProvider`.

## Зависимости

*   `typing.Callable`: Для аннотации типов callback-функций.
*   `SLM.appGlue.DesignPaterns.factory.StaticFactory`: Базовый класс для `DataConverterFactory`.

## Основные компоненты

### 1. Схемы Данных

*   **`GlueObjectFieldDataScheme`**:
    *   **Назначение:** Описывает схему одного поля объекта.
    *   **Атрибуты:**
        *   `name` (str): Имя поля.
        *   `field_type` (type): Тип данных поля.
        *   `default`: Значение по умолчанию для поля.
    *   **Методы:**
        *   `__get__`, `__set__`: Реализуют дескриптор для доступа к значению поля в экземпляре объекта (хотя это не используется напрямую в других частях этого файла).
        *   `GetValue(self, obj_instance)`: Получает значение поля из `obj_instance`. *Примечание: Реализовано только для `str`.*
        *   `SetValue(self, obj_instance, value)`: Устанавливает значение поля в `obj_instance`. *Примечание: Реализовано только для `str`.*
*   **`GlueObjectDataScheme`**:
    *   **Назначение:** Описывает схему целого класса данных.
    *   **Атрибуты:**
        *   `name` (str): Имя класса.
        *   `dataclass` (type): Сам класс.
        *   `fields` (list[`GlueObjectFieldDataScheme`]): Список схем для публичных полей класса.
    *   **Статический метод `scheme_from_cls(cls, *args, **kwargs)`:**
        *   Создает и возвращает экземпляр `GlueObjectDataScheme` для переданного класса `cls`.
        *   Автоматически определяет публичные не-callable атрибуты класса, создавая для них `GlueObjectFieldDataScheme`.

### 2. Преобразование и Маппинг Данных

*   **`DataAdapter`**:
    *   **Назначение:** Базовый адаптер для преобразования данных между типами с использованием зарегистрированных конвертеров.
    *   **Атрибуты:**
        *   `converters` (list[`GlueDataConverter`]): Список доступных конвертеров.
    *   **Методы:**
        *   `Convert(self, data_item, target_type)`: Пытается найти подходящий конвертер в `converters` для преобразования `data_item` к `target_type` (или наоборот) и вызывает его. Возвращает результат конвертации или `None`. *Примечание: Логика поиска конвертера выглядит базовой.*
*   **`DataSchemeMapper`**:
    *   **Назначение:** Копирует данные между двумя экземплярами объектов на основе их зарегистрированных схем (`GlueObjectDataScheme`).
    *   **Атрибуты:**
        *   `schemas` (dict): Словарь, где ключи - это классы данных, а значения - их `GlueObjectDataScheme`.
    *   **Методы:**
        *   `AddSchema(self, schema)`: Регистрирует схему данных для класса.
        *   `map(self, source_instance, target_instance)`: Копирует значения полей из `source_instance` в `target_instance` для всех полей, имена которых совпадают в их схемах. Использует `GetValue` и `SetValue` из `GlueObjectFieldDataScheme`.

### 3. Провайдеры Данных

*   **`DataProvider`**:
    *   **Назначение:** Абстрактный базовый класс для провайдеров данных. Определяет общий интерфейс для доступа к данным из различных источников (БД, файлы, API и т.д.).
    *   **Атрибуты:**
        *   `name` (str): Имя провайдера.
        *   `dataformath` (str): Формат данных, с которым работает провайдер (не используется последовательно).
        *   `dataAdapter` (`DataAdapter`): Экземпляр адаптера для преобразования данных.
        *   `backStores` (list[`DataProvider`]): Список других провайдеров данных, которые могут использоваться как резервные или нижележащие хранилища.
    *   **Методы (многие абстрактные или с базовой реализацией):**
        *   `GetDataItem(self, **kwargs)`: Получить элемент данных (абстрактный).
        *   `GetDataItemFormat(self, dataitem)`: Получить формат элемента данных (не реализован).
        *   `GetDataItemByBackStore(self, storename, **kwargs)`: Получить данные из конкретного `backStore` по имени и конвертировать их с помощью `dataAdapter`.
        *   `SetDataItem(self, item, **kwargs)`: Сохранить/обновить элемент данных (абстрактный, но вызывает `SetDataItem` у `backStores`).
        *   `deleteDataItem(self, item)`: Удалить элемент данных (абстрактный, но вызывает `deleteDataItem` у `backStores`).
        *   `all(self, **kwargs)`: Получить все элементы данных (абстрактный).
        *   `OnDataChange(self, **kwargs)`: Обработчик события изменения данных (вызывает `OnDataChange` у `backStores`).
        *   `InvokeDataChange(self, **kwargs)`: Инициировать событие изменения данных (вызывает `InvokeDataChange` у `backStores`).
        *   `commit(self)`: Подтвердить изменения (вызывает `commit` у `backStores`).

### 4. Конвертеры Данных

*   **`GlueDataConverter`**:
    *   **Назначение:** Абстрактный базовый класс для конвертеров данных.
    *   **Методы (абстрактные):**
        *   `Convert(self, data)`: Преобразование из исходного формата в целевой.
        *   `ConvertBack(self, data)`: Преобразование из целевого формата обратно в исходный.
*   **`ForwardConverter`**:
    *   Наследует `GlueDataConverter`.
    *   Простая реализация, которая возвращает данные без изменений в обоих направлениях.
*   **`damyTostrConverter`**: *(Опечатка в имени: "damy" вместо "dummy")*
    *   Наследует `ForwardConverter`.
    *   **Назначение:** Преобразует объект в строку (`str(val)`) и сохраняет соответствие в словаре `val_str` для возможности обратного преобразования.
    *   **Предупреждение:** Это очень примитивный конвертер, который будет работать некорректно, если строковые представления разных объектов совпадут. Не подходит для общего использования.
*   **`DataConverterFactory`**:
    *   Наследует `SLM.appGlue.DesignPaterns.factory.StaticFactory`.
    *   **Назначение:** Фабрика для создания экземпляров конвертеров по имени.
    *   **Атрибуты:**
        *   `converter_dict` (dict): Словарь для регистрации типов конвертеров по именам. *Примечание: В коде словарь пуст, регистрация не показана.*
    *   **Методы:**
        *   `build(self, converter_name, **kwargs)`: Возвращает экземпляр конвертера по имени из `converter_dict`.

### 5. Селектор Шаблонов Адаптеров

*   **`AdapterTemplateSelector`**:
    *   **Назначение:** Помогает выбрать подходящий "шаблон" (например, класс виджета GUI, другой адаптер) для отображения или обработки элемента данных (`item`) на основе его типа или пользовательских правил.
    *   **Атрибуты:**
        *   `template_map` (dict[type, type]): Словарь, сопоставляющий типы данных с их шаблонами.
        *   `get_template_del` (list[Callable]): Список функций-селекторов. Каждая функция принимает `item` и возвращает шаблон, если он подходит, иначе `None`.
        *   `default_template` (type): Шаблон, используемый по умолчанию, если ни одно правило не сработало.
    *   **Методы:**
        *   `add_template(self, ittype: type, template: type)`: Регистрирует шаблон для конкретного типа данных.
        *   `add_template_selector(self, selector: Callable[[object], object])`: Добавляет пользовательскую функцию-селектор.
        *   `get_template(self, item) -> type`: Определяет и возвращает подходящий шаблон для `item`, сначала проверяя пользовательские селекторы, затем `template_map`, и в конце возвращая `default_template`.

Этот файл закладывает основу для гибкой работы с данными из различных источников и в различных представлениях, хотя некоторые части (особенно `DataAdapter`, `DataProvider`, `DataConverterFactory`) выглядят как базовые или незавершенные абстракции.
