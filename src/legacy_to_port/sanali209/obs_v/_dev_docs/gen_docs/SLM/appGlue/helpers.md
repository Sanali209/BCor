# Helper Functions (`helpers.py`)

Этот файл содержит набор разнообразных вспомогательных функций и класс `ImageHelper` для общих задач, используемых в различных частях приложения SLM.

## Зависимости

*   `zlib`: Для сжатия данных.
*   `io.BytesIO`: Для работы с бинарными данными в памяти.
*   `base64`: Для кодирования/декодирования Base64.
*   `PIL` (Pillow): Для работы с изображениями (`pip install Pillow`). Импортирует `Image`, `ImageFile`.
*   `SLM.files_data_cache.pool.PILPool`: Для эффективной загрузки изображений PIL.
*   `hashlib`: Используется в `ImageHelper.content_md5`.

**Примечание:** `ImageFile.LOAD_TRUNCATED_IMAGES = True` устанавливается глобально, что позволяет Pillow пытаться загружать поврежденные или усеченные изображения.

## Функции

*   **`get_colab_xmlrpc_ngrock_url()` -> str | None:**
    *   **Назначение:** Читает URL из **жестко закодированного пути** на Google Диске (`G:\Мой диск/ngrok_url.txt`), предполагает определенный формат строки, полученной от ngrok в Google Colab, и преобразует его в URL для XML-RPC.
    *   **Возвращает:** Отформатированный URL (например, `https://<id>.ngrok-free.app/RPC2`) или `None` при ошибке (например, файл не найден).
    *   **Предупреждение:** Эта функция очень специфична для конкретной настройки с Google Colab, ngrok и Google Drive. Жестко закодированный путь делает ее непереносимой и хрупкой.
*   **`image_to_base64(image_path: str) -> str:`**
    *   Читает файл изображения по указанному пути (`image_path`).
    *   Кодирует бинарное содержимое файла в строку Base64 (UTF-8).
    *   Возвращает строку Base64.
*   **`base64_to_image(base64_str: str) -> Image.Image:`**
    *   Декодирует строку Base64 (`base64_str`) в бинарные данные.
    *   Открывает бинарные данные как изображение с помощью `PIL.Image.open`.
    *   Возвращает объект `PIL.Image`.
*   **`image_to_zipped_base64(image_path: str) -> str:`**
    *   Загружает изображение по `image_path` с помощью `PILPool.get_pil_image`.
    *   Сохраняет изображение в формате JPEG в `BytesIO`.
    *   Сжимает полученные байты JPEG с помощью `zlib.compress`.
    *   Кодирует сжатые байты в строку Base64 (UTF-8).
    *   Возвращает сжатую строку Base64. Используется для уменьшения размера передаваемых изображений.

## Класс `ImageHelper`

Содержит статические методы для работы с изображениями PIL.

*   **`create_thumbnail_pil(path: str, width: int, height: int) -> Image.Image:`**
    *   Загружает изображение по `path` с помощью `PILPool.get_pil_image`.
    *   Создает миниатюру изображения "на месте" (`image.thumbnail((width, height))`), сохраняя пропорции и уменьшая изображение так, чтобы оно вписывалось в указанные `width` и `height`.
    *   Возвращает измененный объект `PIL.Image`.
*   **`content_md5(img: Image.Image) -> str:`**
    *   Получает сырые байты изображения (`img.tobytes()`).
    *   Вычисляет MD5 хэш этих байтов.
    *   Возвращает MD5 хэш в виде строки. Полезно для проверки идентичности содержимого изображений.
*   **`image_load_pil(path: str) -> Image.Image:`**
    *   Загружает изображение по `path` с помощью `PILPool.get_pil_image`.
    *   Создает и возвращает **копию** загруженного изображения (`.copy()`). Это гарантирует, что исходный объект в пуле не будет изменен, и закрывает файловый дескриптор, если он был открыт.
*   **`image_pil_convert_to_jpg_format(img: Image.Image) -> Image.Image:`**
    *   Проверяет формат изображения (`img.format`).
    *   Если формат не "JPEG", конвертирует изображение в режим "RGB" (`img.convert("RGB")`). Это необходимо перед сохранением в JPEG, если исходное изображение имело альфа-канал (RGBA) или было в другом формате (например, PNG, GIF).
    *   Возвращает исходный или конвертированный объект `PIL.Image`.
*   **`save_image_pil(thumbnail: Image.Image, thumbpath: str):`**
    *   Сохраняет переданный объект `PIL.Image` (`thumbnail`) по указанному пути (`thumbpath`). Формат сохранения определяется расширением файла в `thumbpath` (или может быть указан явно через дополнительные аргументы `save`).
