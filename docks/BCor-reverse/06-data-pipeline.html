6.3 Media Processing Pattern

Download → Hash → Dedup → Store

1. ResourceDownloader fetches file from URL
2. MD5 hash computed (populate_md5_cache.py)
3. Perceptual hash (imagehash.phash) for image dedup
4. DeduplicationManager checks: existing MD5 or phash?
5. If duplicate → DupeDialog (user decision: keep/skip/replace)
6. If new → store to disk, record in SQLite

Metadata Pipeline (ExifTool)

1. MetadataService.read_metadata(fs, path)
2. Spawns exiftool.exe subprocess (thread via asyncio)
3. Returns JSON dict of all EXIF/XMP/IPTC tags
4. Stored as JSON blob in media_files.metadata
