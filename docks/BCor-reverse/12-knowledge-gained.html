12.1 Lessons Learned

LESSON

Async-native Clean Architecture работает

Где: Весь проект

Суть: BCor показывает, что Clean Architecture + DDD + CQRS можно построить на чистом async/await без синхронных прослоек. dishka + bubus + returns образуют когерентный стек.

Применимость: Для новых Python-проектов, требующих testability и модульности.

LESSON

AGM — интересная альтернатива ORM для графов

Где: src/modules/agm/

Суть: AGMMapper предоставляет декларативное GOM (Graph-Object Mapping) с аннотациями @Stored, @Live, @Rel. Identity Map + UNWIND batching + fluent Cypher query builder.

Применимость: Для проектов на Neo4j — лучшая альтернатива neomodel.

LESSON

qasync — работает, но хрупко

Где: src/apps/asset_explorer/main.py

Суть: Интеграция asyncio с Qt через qasync возможна, но требует WindowsLoopManager и осторожности с сигналами/слотами и QProcess.

12.2 Patterns Worth Adopting

PATTERN

Metadata-driven UI Generation

Где: AutoMetadataPanel, SearchConstructor

Суть: AGM-аннотации полей (@Column, @Searchable, @DisplayName) используются для автоматической генерации GUI. Никакого ручного вязания формы к модели.

PATTERN

Stored Field Recalculation

Где: Stored metadata + TaskIQ workers

Суть: Поля, помеченные @Stored, автоматически пересчитываются в фоне при изменении source-поля. Декларативная конфигурация, асинхронное выполнение.

PATTERN

Dishka Provider Scopes

Суть: Scope.APP для singleton'ов (Settings, MessageBus), Scope.REQUEST для per-operation (UoW). Автоматическая DI в обработчиках через параметры.

12.3 Anti-Patterns to Avoid

ANTI-PATTERN

Raw SQL в приложении Clean Architecture

Где: boruscraper/common/database.py

Суть: Сырые SQL-запросы нарушают принцип Clean Architecture. Нет миграций, нет type safety.

ANTI-PATTERN

Toy modules alongside real modules

Где: orders/, ecs/

Суть: Примеры и реальный код в одной директории создают путаницу. Лучше выносить примеры в отдельную папку.
