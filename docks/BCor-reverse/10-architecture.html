10.2 Dependency Rule

Inner layers NEVER depend on outer layers:

  • Domain: src/core/domain.py, src/core/messages.py — чистый Python, ноль внешних импортов
  • Application: src/core/messagebus.py, src/core/unit_of_work.py — зависит от Domain, импортирует bubus/dishka
  • Infrastructure: src/adapters/ — реализует интерфейсы Domain (Repository, UoW)
  • Presentation: src/apps/ — точка входа, зависит от всего

10.3 Pattern Catalog

PATTERN

Event-Driven Modular Monolith

Модули изолированы, общаются только через события. Никаких прямых вызовов между модулями. Каждый модуль — независимый бизнес-домен.

PATTERN

CQRS

Command → 1 handler (изменение состояния), Event → N subscribers (факты). Message наследует bubus.BaseEvent.

PATTERN

Unit of Work + Repository

UoW координирует транзакции и собирает события. Repository абстрагирует хранение. seen-трекинг для domain event collection.

PATTERN

Dependency Injection (dishka)

Автоматическое внедрение зависимостей через параметры обработчиков. Scopes: APP (singleton), REQUEST (per-operation).

PATTERN

Result Monad Railway

BusinessResult = Result[T, E]. Бизнес-ошибки = Failure, инфраструктурные ошибки = исключения. Fail-fast конфиги, fail-safe события.

PATTERN

Config-Driven Module Discovery

TOML манифест → ModuleDiscovery → динамический импорт BaseModule подклассов → DI container.

PATTERN

Strangler Fig (Legacy Migration)

LegacyAllocatorBridge: dishka → legacy Allocator fallback. BridgeAppAction: legacy AppAction → BCor MessageBus.
