2.5 Extracted Concepts

PATTERN

Rich Domain Model (DDD Aggregate)

Где: src/modules/orders/domain.py:Order, src/modules/ecs/domain.py:EcsWorld

Что: Агрегаты содержат бизнес-логику и инварианты (Order.ship() проверяет статус). События порождаются внутри агрегата через add_event().

Зачем: Бизнес-правила инкапсулированы в домене, а не размазаны по сервисам.

PATTERN

AGM Metadata Annotations

Где: src/modules/agm/metadata.py

Что: Декларативные аннотации (@Stored, @Live, @Rel) на полях dataclass определяют поведение: какие поля сохраняются в Neo4j, какие вычисляются на лету, какие связи создавать. UI-метаданные (@Column, @Searchable) управляют отображением.

PATTERN

Polymorphic Node Hierarchy (Asset)

Где: src/modules/assets/domain/models.py

Что: Иерархия Asset → FileAsset, ImageAsset, VideoAsset. Загрузка через register_subclass в AGM Mapper. Поддержка схем (file://, zip://, s3://, ghost://).

PATTERN

ECS Data-Oriented Design

Где: src/modules/ecs/domain.py

Что: Компоненты — плоские dataclass'ы (PositionComponent, VelocityComponent). EcsWorld — агрегат-контейнер со словарём _components[type][entity_id]. События: TickEvent, CollisionDetectedEvent.
