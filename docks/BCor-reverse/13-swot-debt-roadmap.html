13.1 SWOT Analysis

Strengths

  • Чистая архитектура — строгое разделение слоёв
  • Async-native — современный Python 3.12 стек
  • Мощный GOM для Neo4j (AGM)
  • Отличная документация (ADRs, ARCHITECTURE.md, Ddocks)
  • Полный DI-контейнер с dishka
  • Result monad для бизнес-логики

Weaknesses

  • 66% кода — legacy без тестов
  • Недостаток тестов для common/ и apps/
  • Игрушечные модули (orders, ecs) в основной директории
  • Нет CI/CD
  • Raw SQL в бойлерплейте
  • Избыточная сложность конфигурации

Opportunities

  • Извлечь AGM в отдельную библиотеку
  • Добавить REST API (FastAPI) для внешней интеграции
  • Завершить миграцию legacy → BCor
  • CI/CD через GitHub Actions
  • Автоматическая генерация документации (pydoc)

Threats

  • bubus может перестать поддерживаться
  • NATS требует инфраструктуры
  • Привязка к PySide6 (только десктоп)
  • Отсутствие TypeScript/web версии ограничивает аудиторию

13.4 Recommendations

Critical (немедленно)

  • Заменить raw SQL на SQLAlchemy/Alembic в boruscraper
  • Настроить CI/CD (GitHub Actions): ruff → mypy → pytest → coverage
  • Добавить тесты для src/common/ (минимум 80% coverage)

High (следующий спринт)

  • Переместить orders/ и ecs/ в examples/
  • Сгенерировать uv lock-файл
  • Добавить type checking в CI
  • Завершить миграцию imgededupe

Medium (backlog)

  • Извлечь AGM в отдельный пакет (pip install bcor-agm)
  • Добавить FastAPI для внешних интеграций
  • Автоматическая генерация docs через pydoc
  • Publish библиотек на PyPI
