5.4 Async Event Loop Architecture (qasync)

Оба GUI используют qasync для интеграции asyncio с Qt event loop:

WindowsLoopManager.setup_loop()  # Windows-specific event loop policy
app = QApplication(sys.argv)
loop = qasync.QEventLoop(app)
asyncio.set_event_loop(loop)
with loop:
    loop.run_until_complete(amain(app))

Это позволяет coroutine'ам (bus.dispatch, AGM mapper) работать внутри Qt-приложения без блокировки UI.

PATTERN

Async Qt via qasync

Где: src/apps/asset_explorer/main.py:57, boruscraper/main.py

Что: QEventLoop от qasync заменяет стандартный asyncio loop. Позволяет асинхронным корутинам работать внутри Qt-приложения.

Зачем: BCor построен на async/await. Без qasync пришлось бы дублировать синхронные версии всех операций.

PATTERN

Dynamic Metadata-Driven GUI

Где: AutoMetadataPanel, SearchConstructor

Что: UI строится динамически на основе AGM-метаданных полей. SearchConstructor читает схему модели и создаёт соответствующие виджеты (range, date, text, vector).

PATTERN

QProcess Worker Management

Где: WorkerManager в asset_explorer/main.py:298

Что: TaskIQ worker запускается как отдельный процесс через QProcess. stdout парсится для [BCOR_TASK] событий. GUI остаётся отзывчивым.
