# ВАЖНОЕ ЗАМЕЧАНИЕ (IMPORTANT NOTICE)

Эти документы в папке `Ddocks/` являются **концептуальными спецификациями** и **техническим заданием (ТЗ)**, которые использовались для проектирования системы на начальном этапе.

⚠️ В соответствии с принципом **Code First** (Сначала Код), текущая реализация архитектуры в репозитории (в папках `src/core/`, `src/adapters/` и `src/modules/`) **является единственным актуальным источником истины**.

По ходу разработки некоторые технические решения были уточнены или заменены на более современные аналоги:
1. Внутренняя шина на словарях была заменена на специализированную асинхронную шину `bubus.EventBus`.
2. Внедрен DI-контейнер `Dishka` и паттерн Composition Root (класс `System`).
3. Добавлена строгая декларативная конфигурация `pydantic-settings` (Fail-Fast).
4. Тяжелые фоновые задачи маршрутизируются во внешний брокер `NATS` через оркестратор `TaskIQ`.
5. Внедрен слой Observability (Loguru, OpenTelemetry, Prometheus).
