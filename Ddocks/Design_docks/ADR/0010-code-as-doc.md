# ADR 0010: Принцип "Код как документация" (Code-as-Dock)

## Статус
Принято (Accepted)

## Контекст
В монорепозитории BCor документация (Ddocks) часто отстает от интенсивных изменений кода. Устаревшая документация создает технический долг (см. TD-0002) и усложняет онбординг новых разработчиков.

## Решение
Мы внедряем стандарт **"Код как документация" (Code-as-Dock)** для всего современного кода в `src/`. 
1. **Диаграммы в коде**: Все сложные процессы (например, Bootstrap системы или работа MessageBus) снабжаются Mermaid-диаграммами непосредственно в Docstrings классов.
2. **Diátaxis Style**: Docstrings модулей и ключевых классов не просто описывают "Что", но и предоставляют разделы `Example` (как использовать), `Rationale` (почему так сделано) и `Conceptual Overview` (контекст).
3. **Actionable Errors**: Сообщения об ошибках (особенно ImportError) должны содержать готовую команду для их исправления (например, `uv pip install ...`).

## Последствия
- **Положительные**: Документация всегда синхронизирована с фактическим кодом. Разработчик получает всю необходимую информацию, не выходя из контекста IDE.
- **Отрицательные**: Увеличение объема программных файлов. Необходимость настройки рендеринга Mermaid в IDE для полноценного восприятия.
