# ADR 0006: Использование возможностей Python 3.12+ (PEP 695)

## Статус
Принято (Accepted)

## Контекст
С выходом Python 3.12 был представлен новый синтаксис для работы с обобщенными типами (Generics) и псевдонимами типов (Type Aliases) согласно PEP 695. Старый синтаксис с использованием `TypeVar` и `Generic[...]` является более многословным и менее читаемым. Поскольку наш проект ориентирован на современные стандарты и уже использует `uv` с фиксацией версии `requires-python = ">=3.12"`, мы можем полноценно использовать новые возможности языка.

## Решение
Мы официально переходим на использование синтаксиса Python 3.12 для всех новых Generic-классов, функций и псевдонимов типов.

1.  **Generics**: Использование квадратных скобок сразу после имени класса/функции: `class Box[T]: ...` вместо `T = TypeVar("T"); class Box(Generic[T]): ...`.
2.  **Type Aliases**: Использование ключевого слова `type`: `type Point = tuple[float, float]` вместо `Point = tuple[float, float]`.
3.  **Bounds and Constraints**: Новый синтаксис для ограничений типов: `def func[T: str | int](val: T): ...`.

## Последствия
- **Положительные**:
    - Код становится более лаконичным и чистым.
    - Улучшенная поддержка статического анализа (mypy, pyright).
    - Отсутствие необходимости явно объявлять `TypeVar` в начале файла.
- **Отрицательные**:
    - Полная несовместимость с версиями Python ниже 3.12 (уже закреплено в `pyproject.toml`).
    - Необходимость обновления инструментов IDE и линтеров до последних версий.
