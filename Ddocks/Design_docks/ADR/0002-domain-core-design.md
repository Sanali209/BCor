# ADR 0002: Проектирование Domain Core для ImageAnalyze

## Статус
Принято (Accepted)

## Контекст
Нам необходимо выделить чистую бизнес-логику из легаси кода `core/image_scanner.py` в слой Domain Core, чтобы следовать принципам Hexagonal Architecture и обеспечить тестируемость без зависимостей от внешних библиотек (PIL) и файловой системы.

## Решение
1.  **Сущность (Entity)**: Создать `ImageMetadata` как неизменяемый объект данных (dataclass), представляющий результаты сканирования.
2.  **Порт (Port/Interface)**: Создать интерфейс `IImageScanner`, который определяет контракт для сканирования одного файла или директории.
3.  **Слой приложения (Application/Use Case)**: Создать `ScanDirectoryUseCase`, который координирует процесс сканирования, используя порт `IImageScanner`. Это позволит нам тестировать логику сканирования (например, фильтрацию или обработку результатов) с использованием mock-сканера.
4.  **Разделение**: Слой Domain не должен импортировать `PIL` или `os` напрямую. Все специфичные для реализации детали (PIL, Multiprocessing) будут вынесены в адаптеры в Фазе 3.

## Последствия
- **Положительные**: Бизнес-логика полностью отделена от библиотек обработки изображений. Легкое написание Unit-тестов.
- **Отрицательные**: Небольшое увеличение количества файлов и абстракций на начальном этапе.
