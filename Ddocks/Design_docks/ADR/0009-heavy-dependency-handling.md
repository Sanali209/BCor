# ADR 0009: Динамическое управление тяжелыми зависимостями

## Статус
Принято (Accepted)

## Контекст
Ядро фреймворка BCor и общие библиотеки (`src/common`) должны оставаться легкими. Модули ИИ (scikit-learn, umap-learn) и обработки медиа (pyexiv2, exiftool) являются тяжелыми и могут быть не нужны всем пользователям или на всех платформах. 

## Решение
Мы внедряем две стратегии:
1. **Optional Extras**: Все тяжелые зависимости объявляются как `optional-dependencies` в `pyproject.toml`. 
2. **Dynamic Imports**: Внутри кода (в `src/common`) импорты тяжелых библиотек выполняются **внутри** соответствующих методов или классов, используя обертку `try-except` или вспомогательные методы. 

При отсутствии нужной библиотеки возбуждается информативная ошибка `ImportError` с командой для ее установки: `uv pip install ...`.

## Последствия
- **Положительные**: Мгновенный запуск системы; отсутствие ненужных зависимостей в основном пакете.
- **Отрицательные**: Ошибки зависимостей обнаруживаются только в рантайме при попытке вызова соответствующей функции. Дополнительные накладные расходы на локальные импорты при первом вызове.
