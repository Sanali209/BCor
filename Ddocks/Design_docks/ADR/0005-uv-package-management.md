# ADR 0005: Использование uv для управления зависимостями и окружением

## Статус
Принято (Accepted)

## Контекст
С ростом монорепозитория BCor стандартные инструменты управления зависимостями (`pip`, `venv`) становятся недостаточно эффективными для обеспечения быстрой и детерминированной сборки. Нам требуется инструмент, обеспечивающий высокую скорость разрешения зависимостей, надежную фиксацию версий (lockfile) и поддержку современных стандартов Python (PEP 621).

## Решение
Мы внедряем **uv** как основной инструмент управления пакетами и виртуальными окружениями.

1.  **Конфигурация**: Все зависимости проекта описываются в `pyproject.toml` в корне репозитория.
2.  **Фиксация**: Для фиксации графа зависимостей используется `uv.lock`.
3.  **Окружение**: Разработчики используют `uv sync` для синхронизации локального окружения `.venv` с описанными зависимостями.
4.  **Запуск**: Рекомендуется использовать `uv run <command>` для запуска скриптов и инструментов, чтобы гарантировать выполнение в правильном контексте.

## Последствия
- **Положительные**:
    - Экстремально быстрая установка зависимостей.
    - Гарантия идентичности окружений у всех разработчиков и в CI.
    - Упрощение управления python-версиями.
- **Отрицательные**:
    - Необходимость установки `uv` в окружении разработчика.
    - Переобучение привычкам использования `pip` (на переходном этапе).
